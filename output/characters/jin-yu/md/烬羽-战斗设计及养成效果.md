### 设计哲学

烬羽的核心是**延迟引爆（delayed detonation）**——她**不做即时输出**，而是把伤害「先存进敌人身上、再让它自燃炸开」。

普攻 / 特殊技往敌人身上播「**余烬印记**」（一层灼烧 DoT），印记**静静累积**，叠满 8 层**自动引爆**（命中即炸，不靠玩家手动、不靠松手）。引爆是一次按层数结算的火爆 AOE，并开启「**余烬窗**」——窗内她的普攻被强化，攒印记 / 攒资源 / 滚雪球。

于是烬羽的爽点不在「这一刀多疼」，而在**铺得越广、连环炸得越烈**：把印记撒满全场，让一片敌人在同一拍里次第自燃，是她的高光。她逼玩家从「我现在打谁」切换到「我**之前**把印记铺到了哪、它们**什么时候**一起炸」——一种**先布局、后收割**的节奏。

> 一句话：**不即时打疼，而是先播种、待自燃；铺得越广，连环炸得越烈。**

### 核心记忆点

1. **播印记 · 自燃引爆** —— 印记叠满 8 层**自动炸**，不靠手动按键 / 不靠松手。玩家的活是「铺得多、铺得匀」，引爆交给系统。
2. **炉火经济** —— 攒「炉火」资源播印记 / 烧长按，**单向不返**（挣得到、花得掉、不回血），逼玩家节流，不能无脑长按糊脸。
3. **余烬窗** —— 引爆后开一扇强化窗，窗内普攻攒火更快 / 伤害更高 / 形态升级，**引爆喂下一次引爆**，滚雪球。

### 主题风格

「明灭与自燃」——铺印记时极克制（灯火幽幽、印记不起眼地累积），引爆时极炽烈（一片敌人同一拍连环炸开，青焰冲天）。视觉靠「悄无声息地积累」与「轰然连环炸」的强反差建立辨识度，区别于即时爆发型主 C。

### 类似玩法（业界参考）

- DoT 流 / 引燃流的「先铺后炸」节奏感（点燃叠层 → 引爆收割）
- 「种火 → 自燃 → 连环」的延迟结算手感
- （以上为节奏 / 手感参考，机制为原创虚构）

### 推荐配队（《星澜纪》三人队 = 一主双辅）

- **烬羽（主 C · 满层引爆收割）+ 一名铺场 / 控场辅助（聚怪、给烬羽把印记一次铺满一堆）+ 一名增益辅助（火增伤 / 回炉火 / 缩 CD）。**
- 核心出伤点 = 烬羽的**满层连环引爆**；双辅的职责是**制造「能把印记安全铺满一片、再一起炸」的窗口**（聚怪 + 续炉火 + 抬火伤）。配队细节见「陆 · 配队协力」。

### 核心循环手法

- **短轴**：普攻 4 段（播印记 + 攒炉火）→ 短按「引燃」补印记 → 等某个敌人叠满 8 层**自动引爆** → 进余烬窗，窗内普攻继续滚。
- **长轴**：开局短按「引燃」铺底 → 炉火攒够 → 长按「引燃」对一片敌人**猛叠印记 → 连环自动引爆** → 余烬窗内强化普攻继续铺 → 大招「焚天·万烬」把全场印记**单次连环大引爆**收割 + 拉满炉火 + 拉长余烬窗。
- **心法**：永远在「**再铺一层让它满 8 自燃** vs **现在炉火不够要节流**」之间做取舍；引爆不是终点，是**喂养下一轮**的起点。

## 贰 · 核心机制

> 烬羽有两套互锁的系统：**敌方身上的「余烬印记」**（伤害怎么发生）+ **己方的「炉火 / 余烬窗」**（节奏怎么管理）。本章给机制全貌，资源系统的精确数值在「肆 · 资源系统」深入。

### 机制一 · 余烬印记（敌方 DoT 层 · 满层自燃）

- **怎么获得**：烬羽的普攻、短 / 长按「引燃」、联动技、大招命中敌人时，给该敌人**叠余烬印记**（每次命中 +1～多层不等）。
- **印记效果**：每层是一道**灼烧 DoT**（持续掉血），多层 DoT 叠加。
- **🔥 满层自燃引爆**：单个敌人印记叠到 **8 层**（C3 后上限 10）即**自动引爆**——**命中即炸**，不需要玩家手动按引爆键、不需要松开长按。引爆 = 一次按「引爆时层数」结算的**火爆 AOE**（炸到自身 + 周围敌人）+ 开 / 刷新「余烬窗」。
- **设计意图**：把「输出」从玩家手上**挪到敌人身上的计数器**。玩家管「铺」，系统管「炸」——铺得越多越匀，自燃来得越密。

### 机制二 · 炉火（己方资源 · 单向不返）

- **怎么攒**：普攻命中攒炉火、余烬窗内攒更快、完美闪避回一小笔、引爆**不返炉火**（见下「节流」）。
- **怎么花**：长按「引燃」**持续耗炉火**猛叠印记；炉火 ≥ 30% 才能启动长按。
- **🔒 单向不返**：炉火**挣得到、花得掉、绝不回退**——引爆的唯一产出是「开余烬窗」，**不返还炉火**。这是烬羽的**节流阀**：不能无脑长按糊脸，必须靠普攻 / 闪避把炉火攒回来，才有下一次猛铺。
- **设计意图**：给「延迟引爆」一个**资源天花板**。没有节流的话，长按猛叠印记会让自燃失控变成无脑刷屏；单向不返逼出「攒—花—再攒」的节奏。

### 机制三 · 余烬窗（己方强化窗 · 引爆喂引爆）

- **怎么开**：任意一次**引爆**（满层自燃 / 大招大引爆）触发或**刷新**余烬窗。
- **窗内增益**：① 普攻**攒炉火速度↑** ② 普攻 / 印记**伤害↑** ③ 普攻**形态升级**（甩出更宽的扇形焰、播印记更多）。
- **时长有上限**：余烬窗时长**封顶**——连环引爆能**刷新**（续命）但不能无限叠时长，避免「一次铺满全程开窗」破节奏。
- **设计意图**：让**引爆喂下一次引爆**——炸完进窗 → 窗内攒火更快、铺印更多 → 更快触发下一次自燃 → 再刷新窗。滚雪球，但被「窗时长上限 + 炉火单向不返」双重夹住，不会无限膨胀。

### 三系统互锁图（节奏闭环）

```
普攻/引燃 ──播──▶ 余烬印记(敌身, DoT)
   │                    │ 叠满 8 层
   │攒                  ▼
炉火(己方) ◀─不返─── 自动引爆(火爆 AOE)
   │ ≥30% 启动长按       │ 开/刷新
   │                    ▼
长按引燃 ──猛叠──▶ 余烬窗(己方强化窗)
                        │ 普攻攒火↑/伤害↑/形态↑
                        └──▶ 更快铺满 → 下一次自燃(回到顶部)
```

## 叁 · 招式详解

> 招式集按 [《星澜纪》约定 §2](../../../../conventions/example-conventions-gacha.md)：普攻连段（4–5 段）/ 闪避（含完美窗）/ **核心特殊技**（机制载体）/ **终结技**（能量满）/ **切人联动技**（切换上场触发一次）。可选：空中攻击、弹反。每招用「**表现 / 玩法**」双标签描述。预警契约：红光必闪 / 黄光可弹反。

### 1. 普攻 4 段 ·「引魂灯 · 焰火连」

- **表现**：提灯甩出焰火弹，四段递进——横甩 → 撩焰 → 转身扫 → 顿步重砸；末段灯火砸地、击退 + 留一小片火印区。
- **玩法**：每段命中**播 1 层余烬印记** + **攒炉火**；末段**击退** + 落一块**小火印区**（站区内的敌人持续被叠印记）。一套打下来是烬羽「铺底 + 攒火」的主力，**不追求单段疼**，追求把印记匀匀铺出去。
- 定位：普攻 / 播印记 / 攒炉火 / 末段控场。

### 2. 闪避 + 完美闪避 ·「灯影 · 错身」

- **表现**：踏一步灯影错身规避；完美窗内触发「青焰残影」。
- **玩法**：完美闪避窗 6f；完美触发**回一小笔炉火** + 短暂「灯影」状态（可顺势接普攻继续铺）。把**防御操作反哺成攒火**，鼓励贴脸博弈而不是远远苟着等自燃。
- 定位：位移 / 完美闪避 / 回炉火。

### 3. 核心特殊技 ·「引燃」（短按 / 长按）

烬羽的机制载体，一键双形态：

- **短按（免费 · 开局即用）**
  - **表现**：挥灯洒出一束焰火，前方炸开一小片火区。
  - **玩法**：**不耗炉火**，播**少量印记** + 落一块**小火区**。开局 / 炉火见底时的铺底手段，随时可用、零成本。
- **长按（炉火 ≥ 30% 才启动 · 持续耗炉火）**
  - **表现**：引魂灯大放，持续喷吐焰火覆盖前方扇区，灯火越烧越旺。
  - **玩法**：持续**耗炉火**，对覆盖区内敌人**猛叠印记**（每 tick + 层）；铺得快的敌人会在长按过程中**接连叠满 8 层 → 连环自动引爆**。这是烬羽「攒一波炉火 → 一次铺爆一片」的核心爽段。炉火耗尽 / 松手即停。
- 定位：核心特殊技 / 播印记主力 / 连环引爆触发器。

### 4. 切人联动技 ·「引魂入场」

- **表现**：烬羽被切换上场时，提灯横扫入场，洒下一大片青焰。
- **玩法**：切人上场**触发一次**——入场**播一片余烬印记**（覆盖前方一群敌人）+ 落一块**大火区** + **给自身一笔炉火**。一上场就把场子「铺好底、续上火」，无缝接管后续猛铺。**铺场 + 接管**型联动。
- 定位：切人联动技 / 群体播印记 / 续炉火 / 接管开场。

### 5. 终结技 ·「焚天 · 万烬」

- **表现**：引魂灯全数点亮升空，全场余烬同时被点——青焰冲天，一片敌人在同一拍里连环炸开。
- **玩法**：能量满释放——对**全场带印记的敌人**做一次**单次连环大引爆**（无视 8 层阈值，把当前所有印记一次性引爆结算，按各自层数叠加伤害）+ **拉满炉火** + **拉长一段余烬窗**。释放期霸体。这是烬羽的全场收割 + 资源重置键：炸完场 → 满炉火 + 长窗，立刻进入下一轮高强度铺爆。
- 定位：大招 / 全场 AOE / 连环大引爆 / 资源重置。

### 6. 弹反（可选 · 黄光预警）·「灯身 · 격」

- **表现**：以灯身格挡来袭，黄光时机精准 → 反推 + 一束反击焰火。
- **玩法**：对齐**黄光可弹反**契约；完美时机弹反 → 对敌**硬直** + **回一笔炉火** + 给该敌**叠几层印记**。给烬羽一个「正面接招也能转化成攒火 + 铺印」的容错手段。
- 定位：防御 / 弹反 / 回炉火 / 补印记。

### 7. 空中攻击（可选）·「落烬」

- **表现**：滞空甩灯洒火印，落地砸出一圈火区。
- **玩法**：空中**洒火印**（对下方敌人叠印记）+ 落地**小 AOE 火区**。给烬羽一个空中也能继续铺印的衔接，处理浮空 / 高台敌人。
- 定位：空攻 / 浮空衔接 / 播印记。

## 肆 · 资源系统（深入）

> 烬羽有**敌方印记 + 己方炉火 + 己方余烬窗**三套互锁系统，故拎一章深入精确数值（占位值 / 相对量，绝对面板待数值策划定）。

### 系统 A · 余烬印记（敌方 DoT 层）

| 项 | 设定 |
|---|---|
| 载体 | **敌方**身上的叠层（每个敌人独立计数） |
| 每层效果 | 一道灼烧 DoT（持续掉血），多层叠加 |
| 叠层来源 | 普攻每段 +1 / 短按引燃 +少量 / 长按引燃每 tick +层 / 联动入场一次性 +一片 / 弹反 +几层 / 空攻 +层 |
| **引爆阈值** | **8 层**（C3 后 **10 层**）→ **自动引爆**（命中即炸，非手动） |
| 引爆产出 | 按「引爆时层数」结算的**火爆 AOE**（自身 + 周围）+ **开 / 刷新余烬窗** |
| 引爆**不**产出 | **不返炉火**（节流，见系统 B） |
| 印记存续 | **不随时间衰减**（见垃圾箱：曾设衰减、砍掉）；引爆 / 敌人死亡时清空该敌印记 |

> **为什么满层自动引爆而非手动**：手动引爆会把节奏权又交回玩家手上、退化成「叠满→按键」的体力活，且削弱「铺得越广连环越烈」的自动连锁美感。自动引爆让玩家专注「铺」，把「炸」交给系统的计数器——这是烬羽区别于普通点燃流的关键。

### 系统 B · 炉火（己方资源 · 0–100%）

| 项 | 设定 |
|---|---|
| 范围 | 0–100% |
| 攒火来源 | 普攻命中 +小笔 / **余烬窗内攒速↑**（约 ×1.5）/ 完美闪避 +一小笔 / 弹反 +一笔 / 联动入场 +一笔 / 大招**拉满** |
| 耗火去向 | 长按引燃**持续耗火**（每 tick 扣） |
| 长按门槛 | 炉火 **≥ 30%** 才能启动长按引燃 |
| **🔒 单向不返** | 炉火**只增 / 只减、绝不回退**；引爆**不返炉火**——挣得到、花得掉、不回血 |

> **为什么单向不返**：这是烬羽**唯一**的节流阀。若引爆返炉火，长按 → 引爆 → 返火 → 再长按会自我循环、自燃失控成刷屏。单向不返逼出「普攻 / 闪避攒回炉火」的下行节奏，让「猛铺一波」有成本、有间歇。

### 系统 C · 余烬窗（己方强化窗 · 有时长上限）

| 项 | 设定 |
|---|---|
| 开启 / 刷新 | 任意**引爆**（满层自燃 / 大招大引爆）触发或**刷新**窗 |
| 窗内增益 | ① 普攻攒炉火 ×1.5 ② 普攻 / 印记伤害 +30%（C5 后 +时长）③ 普攻**形态升级**（更宽扇形焰 + 播印 +1） |
| **时长上限** | 窗时长**封顶**（如 8s）；连环引爆**刷新**起算点（续命）但**不突破上限** |
| 设计意图 | 引爆喂引爆的雪球引擎，但被「上限 + 炉火不返」夹住，不无限膨胀 |

### 资源节奏闭环（一句话）

**普攻 / 短按引燃攒炉火 + 铺底印记 → 炉火够了长按猛铺 → 一片敌人接连叠满 8 层自燃 → 进余烬窗（攒火↑/伤害↑/形态↑）→ 窗内继续猛铺 → 更密的自燃刷新窗 → 炉火见底（单向不返）→ 回普攻攒火 → 大招「焚天·万烬」全场连环大引爆 + 满炉火 + 长窗，重置进下一轮。**

## 陆 · 配队协力

> 《星澜纪》三人队（一主双辅）+ 切人联动技 + 共享「协力能量」槽。烬羽是**主 C**，双辅服务她的「铺满 → 连环引爆」窗口。

### 队伍模型

- **主 C：烬羽** —— 核心出伤 = **满层连环引爆**收割（短轴自燃 + 长轴猛铺 + 大招全场大引爆）。
- **辅助一（铺场 / 控场）** —— **聚怪 + 站桩窗**：把散开的敌人拢成一堆，让烬羽一次长按 / 一发联动就能把印记**铺满一整片**（印记铺得越集中，连环自燃越密）。
- **辅助二（增益 / 续航）** —— **火增伤 + 回炉火 + 缩 CD**：抬烬羽的火系伤害乘区、帮她把「单向不返」的炉火补回来、缩短引燃 / 大招间歇。

### 协同链路

1. **铺场**：辅助聚怪 → 烬羽「引魂入场」联动一发铺一片印 + 给自己续炉火。
2. **猛铺**：增益辅助续炉火 → 烬羽长按引燃对一堆敌人猛叠印 → 连环自燃。
3. **收割**：协力能量满 → 烬羽大招「焚天·万烬」全场连环大引爆 + 满炉火 + 长窗。
4. **正反馈**：**烬羽每次引爆额外攒一笔协力能量** → 连环引爆 = 协力能量快速回满 → 更快轮到下一次大招 / 队友切人协力。「铺爆一片」既出伤、又喂协力，双重收益。

### 协力能量正反馈（设计点）

把「引爆」和「协力能量」绑定——**引爆越多、协力回得越快**。这让烬羽的「延迟引爆」节奏天然驱动整队的协力节奏：她铺爆一片 → 协力满 → 队友切人放协力技 / 烬羽放大招 → 再开一轮。配队的爽点和烬羽个人的爽点同频。

## 柒 · 数值建模

> 🔴 **硬标准**：5 张图 + 每图「怎么算 · 为什么 · 影响」三段注解 + 平衡自检表。建模口径：《星澜纪》乘区式（`技能 × 增伤 × 暴击`，守方乘区归一化为 1），**本章数值为相对指数 / 倍率**（已归一化），**绝对面板待数值策划定**。PVP 临界点须用项目真实公式复核（见 [numbers-and-progression.md § 十](../../../../references/numbers-and-progression.md)），别用简化式拍板。

### 图① · 命座 C0–C6 累计强度曲线

| 命座 | C0 | C1 | C2 | C3 | C4 | C5 | C6 |
|---|---|---|---|---|---|---|---|
| 累计指数 | 100 | 106 | 122 | 130 | 138 | 146 | 205 |
| 逐级增量 | — | +6% | **+15%** | +6.5% | +6% | +6% | **+40%** |

- **怎么算**：每命座效果折算成对「循环 DPS」的增益 %，在 C0 = 100 基准上**累乘**成累计指数（乘区式，守方乘区归一化为 1）。
- **为什么这么算**：C2 / C6 做质变跳点（付费拐点 + 鲸鱼勋章），其余平滑补，符合 5 段付费哲学。
- **影响**：205 ÷ 100 = **满命 +105%**，落 +80~120% 健康带正中。质变跳点（C2 +15% / C6 +40%）明显高于平滑级（C1/C3/C4/C5 约 +6%），曲线有「两个台阶」的付费诱导感，但 C0 已完整可玩。

### 图② · 逐级边际（分档）

| 档 | 命座 | 边际 | 性质 |
|---|---|---|---|
| 入门 | C1 | +6% | 平滑数值 |
| **质变拐点** | **C2** | **+15%** | 解锁主动引燃（新操作维度） |
| 数值 | C3 | +6.5% | 印记上限 +2 + 引爆系数 |
| 铺场 | C4 | +6% | 火区领域化 |
| 循环 | C5 | +6% | 窗时长 + 炉火攒速 |
| **机制** | **C6** | **+40%** | 野燔（全场连环） |

- **怎么算**：取图①相邻命座累计指数比 − 1，得每级边际增量 %。
- **为什么这么算**：用「分档」看跳点是否落在该落的地方——付费拐点（C2）和鲸鱼勋章（C6）应是边际**局部峰值**，平滑级应低且接近。
- **影响**：C2（+15%）、C6（+40%）两个峰值清晰，中间档（C1/C3/C4/C5 约 +6%）平稳 → **质变跳点合理**，不存在「平滑级比拐点还肥」的倒挂。C6 +40% 是全图最大跳，符合「机制变质 = 鲸鱼专属」。

### 图③ · 增伤区稀释（余烬窗 +30% 在不同面板下的边际）

| 面板已有增伤 | 0% | +60% | +100% |
|---|---|---|---|
| 余烬窗 +30% 的**实际边际** | +30% | ≈ +18.75% | ≈ +15% |

- **怎么算**：《星澜纪》增伤是**单一加法 bucket** `(1 + Σ增伤)`，自带增伤（余烬窗 +30%）实际边际 ≈ `30% ÷ (1 + 已有面板增伤%)`（见 [§ 四](../../../../references/numbers-and-progression.md)）。
  - 面板 0%：30 ÷ 1.0 = **+30%**
  - 面板 +60%：30 ÷ 1.6 ≈ **+18.75%**
  - 面板 +100%：30 ÷ 2.0 = **+15%**
- **为什么这么算**：提醒——烬羽的余烬窗 +30% **会被面板增伤稀释**，面板越肥越不值钱。所以**别按名义 +30% 估强度**，也别让玩家硬堆增伤词条。
- **影响**：烬羽走「**印记叠层 / 引爆层数 / 暴击**」赛道（伤害主要来自「叠多少层 × 引爆系数 × 暴击」，不靠自堆增伤）。余烬窗 +30% 当「锦上添花」而非主乘区，规避加法 bucket 天花板（× ≤ ~1.5）。属性投放推荐：暴击赛道（CR:CD=1:2）+ 攻击力 + 够用充能，**不主堆增伤**。

### 图④ · 场景伤害梯度

| 场景 | 相对强度 | 说明 |
|---|---|---|
| 群战（多目标） | **100%**（基准） | 印记铺一片 → 连环自燃 / 大招全场大引爆，烬羽主场 |
| 小怪 / 精英混合 | ~85% | 仍能铺多个，连环略减但健康 |
| 单 Boss（单体） | **~50%** | 只有一个计数器，无连环、靠单体反复叠满 8 层引爆 |
| PVP（对人） | ~60%（相对值） | 单 / 少目标 + 印记需时间叠满，节奏被压；满命梯度须真实公式复核（见自检） |

- **怎么算**：以「群战多目标连环引爆」为 100% 基准，按可铺印记的目标数 / 是否触发连环，折算其他场景相对强度。
- **为什么这么算**：火系 AOE 主 C 的**定位取舍**——烬羽明确是「越多目标越强」的群战收割型，单体是短板。
- **影响**：单 Boss ~50% 落在 §五「群伤 C 单体 40~55%」健康带（不残废但明显是短板，符合定位）。**配队 / 关卡设计应给烬羽多目标场景**；纯单体 Boss 战她不是首选主 C。这是**主动设计的取舍**，不是失衡。

### 图⑤ · 机制专属曲线 = 引爆收益（印记层数 → 引爆倍率）

| 引爆时层数 | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 | （C3 后）9 | 10 |
|---|---|---|---|---|---|---|---|---|---|---|
| 引爆倍率 | 1.0× | 1.4× | 1.8× | 2.2× | 2.6× | 3.0× | 3.4× | **3.8×** | 4.2× | **4.6×** |

- **怎么算**：引爆伤害 ≈ `基础 × (1 + 0.4 × (层数−1))`，**线性**——1 层 1.0×，每层 +0.4×；正常自燃在 8 层引爆 = **3.8×**；C3 提上限到 10 层 = **4.6×**。
- **为什么这么算**：引爆收益**线性随层数**——「铺得越满、炸得越狠」要可预期、无突变（区别于苍岚 4 层一闪的非线性质变；烬羽的质变在「连环 / 全场」而非「单次倍率跳」）。C3 抬上限 = 把单次引爆天花板从 3.8× 抬到 4.6×。
- **影响**：满层引爆 ≈ 3.8×（C3 后 4.6×）对应单次自燃的「值得」感；但烬羽真正的爆发在**连环**——一片敌人各自 3.8× 同拍炸 + 大招全场大引爆，叠起来远超单体。引爆段相对普攻段约 **8~12×**（对应 §五「爆发 5~15×」健康带）。线性曲线让玩家能心算「铺到几层炸划算」，强化「布局」决策。

### § 五健康自检表

| 维度 | 健康基线 | 烬羽 | 判 |
|---|---|---|---|
| 满命总梯度（C0→C6） | +80~120% | **+105%** | ✅ |
| 逐级 | 累计严格递增 + 拐点跳 | 100 < 106 < … < 205，C2/C6 跳点 | ✅ |
| C0 完整 | 必须 | 印记 / 自燃 / 炉火 / 窗 核心循环 C0 即完整 | ✅ |
| 爆发（引爆 / 大招段） | 5~15× | 引爆段 ≈ 8~12× 普攻段 | ✅ |
| 增伤区膨胀 | × ≤ ~1.5 | 余烬窗 +30% 被稀释、走叠层 / 暴击赛道，≈ 1.1~1.3 | ✅ |
| 暴击 | CR:CD 1:2 不溢出 | 遵守（推荐 65% / 130%） | ✅ |
| 场景梯度（群伤 C 单体） | 40~55% | 单 Boss ~50% | ✅ |
| 养成物套装 | 4pc ≈ 质变命座量级 | 「燎原」4pc 引爆额外播 2 层（条件 +30~50%） | ✅ |
| PVP 满命 / 0 命（有 PVP） | ≤ ~1.8× | 原始伤害梯度 2.05× 逼近上限；HP/防御不随命座涨，有效战力比 ≈ 1.65× | ⚠️ 临界 |

- **PVP 比怎么算**：原始伤害梯度 = 累计指数比 205 ÷ 100 = **2.05×**；但 PVP 战力还含 HP / 防御（不随命座涨），把伤害提升摊进总战力后**有效比 ≈ 1.65×**（示例估算）。2.05× 看着越 1.8× 上限、有效 1.65× 又在区内 —— 正因临界，**必须用项目真实 PVP 公式跑，别用简化式拍板**。
- **闭环结论**：除 PVP 临界一项（需真实公式复核），其余 8 维全部落健康带。烬羽走「叠层 / 引爆 / 暴击」赛道规避增伤稀释，群战主场 / 单体短板是主动取舍，命座 +105% 健康梯度经建模验证落正中。

> 🔎 **本完整稿暴露的方法论价值**：gacha + PVP 语境下，§五 PVP 阈值主动拦下一个临界点（满命伤害梯度 2.05× 逼近 1.8× 上限），提示必须用**项目真实 PVP 公式**复核——正是「用真实公式跑、别用简化式」的用例。

### 元规则命中

- **元规则（Snapshot vs Dynamic）**：印记 DoT / 引爆结算口径需明确（建议引爆按「引爆瞬时层数 + 当时增伤」结算，dynamic）。✅ 已在「肆」标注。
- **元规则（资源条显式化）**：炉火条 / 余烬窗计时 / 印记层数全部 HUD 显式化（含「将燃」高亮）。✅ 见「捌·四」。
- **元规则（防国民老婆 / buff 不破乘区）**：烬羽是主 C 非辅助，增伤走加法 bucket（被稀释、不独立乘区），不存在「独立乘子破平衡」。✅
- **元规则（stack cap）**：印记有 8（/10）层上限 + 引爆即清；炉火 0–100% 封顶 + 单向不返；余烬窗时长封顶。三处都有 cap，无「无限叠加」。✅
- **元规则（养成 0 命完整）**：C0 核心循环完整，C2/C6 是惊喜不是必备。✅

### 垃圾箱（废弃方案存档）

> 留痕设计取舍，方便日后追溯 / 给团队看踩过哪些坑。

- **「余烬印记会随时间衰减」** —— **砍掉**。实测心理压力太大：玩家铺到一半发现前面的层数在掉、变成「跟时间赛跑」，节奏憋屈、不敢慢慢布局。改成**印记不衰减、只在引爆 / 敌死时清空**，把「铺」的决策权还给玩家。
- **「引爆手动触发（按键引爆）」** —— **砍掉**。手动引爆把节奏退化成「叠满 → 按键」的体力活，且削弱「铺得越广、连环自动炸」的连锁美感。改成**满 8 层自动引爆**，玩家专注铺、系统管炸。（注：C2 才以「主动引燃」形式**有限度**地把「提前抢炸」作为养成解锁的进阶维度放回来，而非 C0 默认。）
- **「引爆返还炉火」** —— **砍掉**。返炉火会让「长按 → 引爆 → 返火 → 再长按」自我循环、自燃失控成无脑刷屏。改成**炉火单向不返**，逼出「普攻 / 闪避攒回炉火」的下行节奏，给猛铺一波加成本、加间歇。
