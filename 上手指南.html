1这是什么

把"设计一个动作游戏角色 / Boss"沉淀成一套可复用的方法论，背后配一座行业知识库。它最关键的特点是 —— 不预设任何一套战斗体系。
🟢 方法论 + 行业知识库（可迁移，跟着你走）

设计流程、5 维质量标尺、46 条元规则、跨游戏借鉴库、角色原型 / Boss 范式、打击感 / 数值 / Boss AI 工程基线……

这部分是所有项目通用的，开箱即用。
🟠 你项目的战斗设定（每个项目自己定义）

招式集、视觉预警契约、受击 / CC 分类、分类轴、帧率、命名、配队、养成……

本方法论不强加这些（不假设有"固定 9 招 / 红光必闪 / 协力技 / 命座"），而是引导你在 conventions/TEMPLATE.md 里把你项目自己的约定填好。
▲ 核心理念：通用方法论替你把活干；项目专属设定由你定义，互不绑架。

正因为这条分界，同一套 skill 能服务任何 ACT 项目——二次元抽卡手游、单机硬核动作、横版清关都行；换个项目，只要重填一份"约定"即可。

2核心优点

🧩
项目无关，不强加体系

战斗设定由各项目自定义；方法论只提供"设计空间菜单 + 留空模板 + 中性示例"，绝不把某家的"9 招 / 红光 / 命座"塞给你。

🤖
任意 agent 开箱即用

纯 markdown 知识库 + 可粘贴 system prompt，Claude Code 当 skill、GPT / Claude.ai / Cursor 当上传文件，都能跑。

📑
审阅 / 落地分离

设计稿（给人评审"好不好"）与生产资源表（给美术 / 程序"照着做"）分两套产出，互不淹没。

🙋
默认人审 + 可选全自动

默认在关键决策点停下来跟你一起定；你说"完全出一版"就一口气出全套 + 标好假设。

📚
厚实的行业知识库

角色原型、Boss 范式、46 条元规则、70+ 游戏借鉴、打击感 / 数值 / Boss AI 工程基线 —— 设计有据可依，不拍脑袋。

🎁
自带模板 + 完整示例

审阅级 HTML 模板 + 资源表模板 + 三个从零做好的示例（角色「苍岚」分稿、角色「烬羽」完整稿、Boss「孤鸦」），照葫芦画瓢。

4能做什么

角色 / Boss 完整设计 + 效果养成专项 × 两个产出方向。

📄 审阅级 HTML 长什么样

一份可交互的单文件评审稿 —— 具体内容随设计对象而变，大致是：

🧍
角色

核心机制 + 招式卡、动作帧、特效需求、连招 等

👹
Boss

阶段表、技能详情、AI & 连招策略、金 / 紫预警标注 等

📈
效果与养成

技能附带效果、命座 / 养成物、属性投放、数值图示 + 算的过程 等

三类都自带同一套协作交互：📝 评审备注 · 📜 改动留痕 · 🖼️ 贴图贴视频 · 📤 一键导出回传 · 两级目录 · 🔍 正文图表点击放大（按界面 fit 完整显示，宽图撑满 / 高图触顶自动等比缩、都不被裁，滚轮再放大看细节）—— 给团队评审、迭代到定稿。

看实例 → 烬羽（角色 · 完整稿） · 苍岚（角色 · 分稿） · 孤鸦（Boss） · 苍岚 · 效果与养成

🧍 角色设计稿 = 分稿 / 完整稿两种规格

同一个角色，按节奏需求选一种规格出（用户明示就按明示，没明示 AI 会先问一句再动手）：

📑 分稿 —— 战斗设计稿 + 养成稿分开

「先出战斗设计稿、之后单独再出养成稿」的分步节奏，各自精简。样板：苍岚（苍岚-战斗设计 + 苍岚-养成与效果 两份）。

📘 完整稿 —— 一份到底 9 章全

「要一份完整的」的一站式节奏，含数值建模 / 配队 / 资源清单全部内容 + 操作 / 资源 / 数值三类图。样板：烬羽（烬羽-战斗设计及养成效果 一份）。

▲ 文件命名按稿件内容统一规范：角色名-战斗设计 / 角色名-养成与效果 / 角色名-战斗设计及养成效果 / 角色名-资源需求。「审阅稿」是稿件的协作形态（所有设计稿都自带这套交互），不进文件名。

6两种协作模式

战斗设计需要人拍板。默认替你把关；显式要求时才全自动。
🙋 默认 · 人审协作

在 3 类重大决策点必停，和你一起定：

① 设计输入（中文名 / 定位 / 机制方向，必问）
② 高层框架（"这个方向行不行"）
③ 定稿前（草稿改到满意、你说"定了"才转实际产出）

铁律：设计必须经人工审核 + 协作改定，才进入实际产出。
🤖 可选 · 全自动出稿

当你明确说「你完全出一版」/「全自动出」/「自己出全套不用问我」时：

· 不在决策点停，一口气出全套
· 该问的设计选择自己合理推测拍板
· 产出里集中列出"替你做了哪些假设"

产出标注「自动草案 · 投产前仍建议人工过一遍」。

10常见问题

它会不会把某家公司的战斗体系强加给我？▸

不会。这正是它的设计初衷——招式集、视觉预警、受击分类、分类轴、命座、配队等全部由你项目自定义（写在 conventions/）。知识库里凡是具体体系都明确标注为"示例 / 项目自定义"，不作默认。

只有 Claude 能用吗？GPT 行不行？▸

任意 agent 都行。纯 markdown + 可粘贴 system prompt，不依赖任何 agent 的私有机制。GPT / Claude.ai / Cursor 见 USAGE.md。Claude Code 可作为 skill 自动触发。

设计稿一定要人审吗？能让 AI 全自动出吗？▸

默认人审协作（关键处停下来一起定，再产出）；但你明确说"完全出一版"时可全自动出全套 + 列出所做假设。即便全自动，产出也标注"投产前建议人工过一遍"。详见第 ⑥ 节。

资源 xlsx 在非 Windows 上能生成吗？▸

xlsx 生成脚本（scripts/，PowerShell + Excel COM）仅 Windows + Excel / WPS。非 Windows 用通用的 markdown 资源表（templates/blank-resource-requirements.md）即可，内容一样，只是格式。

自带的示例（苍岚 / 烬羽 / 孤鸦）会污染我的项目吗？▸

不会。它们是放在 output/ 下、明确标注"示例·可删"的演示产出。不需要直接删掉三个文件夹即可。
