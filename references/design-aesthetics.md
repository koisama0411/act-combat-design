# 设计审美 + 战斗节奏理论

让设计者具备"判断设计好坏"的标尺,不只是"按格式填空"。

---

## 一、5 维质量标尺

每个角色/Boss 设计完后,**逐维度自评 1-5 分**,总分 25 分。低于 18 分要重做。

### 维度 1:招式可读性(Telegraph & Clarity)

玩家能不能**看一眼就知道**这招是什么、要怎么应对?

| 分数 | 表现 |
|---|---|
| 1 分 | 招式 telegraph 模糊,玩家完全靠"背板" |
| 2 分 | 有视觉前摇但不区分(所有招都一样的红光) |
| 3 分 | 视觉前摇清晰,但应对方式单一 |
| 4 分 | 颜色语义化(红/黄/蓝),应对方式 2-3 种 |
| 5 分 | 多色 telegraph + 对应解法分类 + 设计师有"教学意图" |

**测量方法**:让一个**没玩过这角色**的新玩家看战斗视频,问"你看到这招会怎么应对?",答得上 = 5 分,答不上 = 1 分。

**业界顶尖**:战神 Ragnarök 红/黄/蓝;只狼危技刺/横/抓;剑星红/黄/蓝/紫。

---

### 维度 2:操作正反馈(Feedback)

玩家按下按键后,**多快收到反馈?反馈多强?**

| 分数 | 表现 |
|---|---|
| 1 分 | 按键后没反馈或反馈延迟严重 |
| 2 分 | 有反馈但反馈是"通用"(所有招都一样的命中音效) |
| 3 分 | 反馈针对招式差异化(轻 vs 重 vs 暴击) |
| 4 分 | 反馈 + 屏幕震动 + Hit-stop + 镜头摇晃,多层叠加 |
| 5 分 | 顶级:hit-stop 帧级精准 + 残影 + 音效叠加 + 振动反馈 |

**测量方法**:看战斗视频时把音乐关掉,**单凭画面**能不能感受到打击感?

**业界顶尖**:鬼泣 5(白金式 hit-stop) / 黑神话(精准帧反馈) / 战双帕弥什(二游 ACT 顶级反馈)

---

### 维度 3:决策点密度(Decisions per Minute)

玩家在战斗中**每秒做多少有意义的决策**?

| 分数 | 表现 |
|---|---|
| 1 分 | 战斗是"按 ABCD 循环",几乎没决策 |
| 2 分 | 偶尔决策(选 1-2 个技能怎么放) |
| 3 分 | 每 5-10 秒一个决策(看招辨型 + 选反击方式) |
| 4 分 | 每 2-3 秒一个决策(切形态/切角色/选 build) |
| 5 分 | 几乎每秒都在决策(石头剪刀布 / 多形态切换 / 资源管理) |

**测量方法**:数 60 秒战斗视频中**玩家主动选择的次数**(选哪招 / 选哪态 / 选哪个反击)。

**业界顶尖**:只狼(每秒决策"弹反/见切/跳") / 永劫无间(连击/蓄力/振刀三方博弈) / FF16(30 秒切 Eikon)

---

### 维度 4:记忆点辨识度(Iconicity)

战斗结束后,玩家能记住**几个具体细节**?能跟别人讲清楚这角色/boss"独有的是什么"?

| 分数 | 表现 |
|---|---|
| 1 分 | 玩完就忘,讲不出独有 |
| 2 分 | 记得"高伤害" / "颜值高",讲不出机制 |
| 3 分 | 能讲出 1-2 个招式名 |
| 4 分 | 能讲出"独家机制"(如"她的特殊技标记敌人") |
| 5 分 | 玩家社区给招式取外号(如"水鸟乱舞"),做 meme,做二创 |

**测量方法**:玩家社区(B 站/贴吧/TapTap)有没有这角色/boss 的**专属梗**?

**业界顶尖**:玛莲妮亚水鸟乱舞 / 苇名一心雷返 / Magnamalo 怨虎龙 / 参议员 Armstrong "It Has To Be This Way"

---

### 维度 5:耐玩度(Replayability)

通关后玩家**还想再玩**这角色/boss 吗?

| 分数 | 表现 |
|---|---|
| 1 分 | 通关一次就够,不想再碰 |
| 2 分 | 有挑战模式但是"重复" |
| 3 分 | 有难度分级 / 评分系统 / 不同 build 玩法 |
| 4 分 | 多套 build / 多周目 / 不同玩家视角 |
| 5 分 | 数百小时不腻,有"高玩流" / 速通文化 / 多周目体验 |

**测量方法**:游戏发布 1 年后,**B 站这个角色/boss 的视频还在更新吗**?有没有"挑战 X 流通关"内容?

**业界顶尖**:艾尔登玛莲妮亚(挑战速通文化) / NieR Automata(多周目结构) / MH 系列(数千小时不腻)

## 二、战斗节奏理论

### 1. ACT 的 4 种节奏感

| 节奏 | 特征 | 代表 |
|---|---|---|
| **高速流水** | hit-stop 密集 + combo 编织 + 高 APM | 鬼泣 / 猎天使 / MGR |
| **重锤呼吸** | 招式之间留停顿 + 重量感 + commitment 蓄力 | 战神 / 黑神话(劈棍) / 怪物猎人(大剑) |
| **拼刀博弈** | 高频弹反 / 见招拆招 / 心理博弈 | 只狼 / 永劫无间 / 卧龙 |
| **指令策略** | 半实时 + ATB / 资源管理 + 协同 | FF7 Rebirth / 终末地 / 异度神剑 3 |

### 2. 心流曲线(Flow Curve)

战斗过程要有"压迫节拍 vs 放松节拍"交替——纯压迫疲劳,纯放松无聊。

```
开战(放松,试探)
↓
小高潮(boss 出招要应对)
↓
缓冲(boss 后摇 + 玩家追击)
↓
中高潮(boss 阶段切换 / 大招)
↓
长缓冲(Stagger 输出窗口 + 高伤倍率)
↓
压力上升(残血 boss 招式更猛)
↓
顶点(★ 出圈招)
↓
结束(KO + 战利品奖励)
```

**典型时长**:小怪 30 秒-2 分钟 / 精英 2-5 分钟 / boss 5-15 分钟 / Raid 20-60 分钟。

### 3. 操作密度的"节奏锚点"

战斗中需要**清晰的节奏锚点**让玩家"听到节拍",不是"自由弹奏"。

**节奏锚点的形式**:
- **音效锚点**:每招命中有独特音效(只狼"叮叮当当"刀剑碰撞)
- **视觉锚点**:每招命中有独特画面(hit-stop / 闪光 / 残影)
- **资源锚点**:每招消耗/获得特定资源(攒能/释放,周期性)
- **节拍锚点**:配合 BGM(剑星 Tachy 配合节拍 / 灵游记)

### 4. 难度曲线(Difficulty Curve)

战斗设计的难度不应该"一直高",而是**波浪上升**:

```
试探期(易) → 学习期(中) → 挫败期(难) → 突破期(顿悟) → 终局(集大成)
```

- **试探期**:让玩家体验角色/boss 基础玩法,不挫败
- **学习期**:逐步引入新机制,玩家"会但不熟"
- **挫败期**:**短暂高难度让玩家失败 1-3 次**,产生"我要打过你"的动机
- **突破期**:玩家"悟"了——找到打法/build,产生强烈成就感
- **终局**:综合考核(类似 G 型 boss 范式),让玩家用上全部学过的

⚠️ 没有挫败期 = 没有突破期 = 没有成就感。**"无脑通关"的设计是失败的设计**。

## 三、设计审美的"硬性原则"

### 原则 1:任何机制都要"可读"

如果玩家**看不出**这招在做什么,这机制就失败。哪怕逻辑再精妙也没用。

### 原则 2:数值是设计的最后一笔

机制确定后才调数值。**反例**:为了平衡而强行让某机制变弱,破坏机制本身的乐趣。

### 原则 3:不要为了"深度"而堆机制

机制不是越多越好。3 个互相协同的好机制 > 10 个孤立的机制。

### 原则 4:玩家的"挫败感"和"成就感"必须配对

挫败感(短期负面)是为了**激发**成就感(长期正面)。如果挫败感不能转化为成就感,设计就是"折磨玩家"。

### 原则 5:每个角色/Boss 都要有"可被讲述的故事"

机制本身要能**对应一个心理图像**:
- ❌ "她的特殊技伤害 5000"
- ✅ "她是个变魔术的小姑娘,每次出招前都要先抽张牌"

### 原则 6:容错与挑战并存

新手按一键能打出帅气连段(One-button heroism,FF16 哲学)+ 高玩能挖出 DMC 级 cancel 链。

**反例**:只对高玩友好(rejected by 大众) / 只对新手友好(rejected by 核心玩家)。

### 原则 7:命名是设计的灵魂

机制名/招式名要**让玩家想用**(参见 [元规则](meta-rules.md) Step 5)。

### 原则 8:设计哲学"一句话"的撰写准则

设计文档开篇的"一句话设计哲学"(角色/Boss 的灵魂概括),按 4 条准则写:

1. **从对象侧描述,不从玩家侧** —— 主语是这个角色/Boss 自己,不是"玩家瓦解 XX"。✅"凭借两件高科技装备居高临下,每件装备都内藏弱点" / ❌"玩家瓦解 Boss 的装备"
2. **严肃,不中二** —— 不要"装备越炫弱点越致命"这种破折号中二尾巴;陈述句直给精髓
3. **说人话,不书面术语** —— "装备代差"这类术语太书面,换成"高科技装备";让非策划也一眼懂
4. **全场景通用,不绑特定结果** —— 多阶段/多场战的对象,一句话要覆盖全部,不能只贴某一场的结局(如"退守机器人"只贴第二场,不通用)

> 🚨 这条一句话往往要打磨好几轮(实战中一个多形态 Boss 的一句话哲学打磨了 6 版才定稿)。**先用 4 准则自检,能省好几轮来回**。

## 四、自评模板

设计完一个角色/Boss 后填这个模板:

```
角色/Boss 名:_____
设计完成日期:_____

【5 维评分】
招式可读性:_/5 (理由:___)
操作正反馈:_/5 (理由:___)
决策点密度:_/5 (理由:___)
记忆点辨识度:_/5 (理由:___)
耐玩度:_/5 (理由:___)
总分:_/25

【元规则命中数】
命中:_____, _____, _____... (列编号,见 meta-rules.md)
未命中但可改进:___

【节奏类型】
属于哪一类节奏感?___

【一句话讲述】
这个角色/Boss 是"___"(用一个比喻形容)

【玩家会取的外号】
预测玩家会叫这角色/招"___"

【难度曲线】
试探期 / 学习期 / 挫败期 / 突破期 / 终局 这 5 段都有吗?___
```

填完后,总分 < 18 分或元规则命中 < 4 条 → **重设计**,不是"加机制凑数"。
