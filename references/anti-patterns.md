# 设计反模式(避坑清单)

业界 25+ 款 ARPG 调研中识别出的**典型设计反模式**——设计时严防出现。

⚠️ 每条反模式后跟"业界踩坑案例"和"避坑方法"。

---

## 反模式 1:单一机制收益过高 → 其他玩法贬值

**症状**:某个机制(如完美弹反、特定连段、某个 Build)收益太高,**其他玩法变得"不值得用"**。

**业界案例**:
- **卧龙·苍天陨落化解**:化解一键通吃 + 收益过高(掉对方气势 + 涨自气势 + 触发处决)→ 玩家社区诟病"不化解就是傻"
- **FF16 Limit Break + Stagger 滚雪球**:开 Limit Break 时 boss 被 staggered = 翻 4 倍输出,部分玩家批评"按 cooldown 就行"

**避坑方法**:
- 设计强力机制时**必须加副作用 / 频率限制**(如卧龙气势槽风险:化解失败 - 气势)
- 玩家有"多种主流玩法",而不是"一种最优解"
- 数值上让"次优玩法"也有 70%+ 的效率,不是 30%

---

## 反模式 2:蓄力期间玩家无操作 → 等待感强

**症状**:角色需要长按蓄力或大招前摇,**期间玩家什么都不能做**——纯等待。

**业界案例**:
- 某长按蓄力削韧的近战角色(初版):蓄力期间是防御 Pose + 自身 Buff,但玩家**只能按住等**
- 经典反例:某些 JRPG 大招读条 5 秒

**避坑方法**:
- 蓄力期间加"博弈"——被击触发反弹判定(类黑神话识破回棍势)
- 蓄力期间可走位 / 切其他招(类 MH 大剑跑动蓄力)
- 蓄力时间分段(类 Hades 2 Ω 系统普通版/蓄力版二级)

---

## 反模式 3:大招太长 + 不可被打断 → 节奏停滞

**症状**:大招动画 8 秒以上 + 期间无敌 + 不能被打断,玩家**看动画时无聊**。

**业界案例**:
- 早期 FF 系列召唤兽大招读条 + 召唤兽演出 30 秒
- 某些 MMO 大招演出过长

**避坑方法**:
- 大招动画 < 5 秒(剑星 Tachy 持续短,反复触发)
- 演出期间加 QTE / 玩家可操作元素(FF7 Rebirth Synergy Abilities 全员同时大招但可加碎片化操作)
- 大招换成"持续形态 5-10 秒"(类猎天使 Demon Slave),玩家有可操作空间

---

## 反模式 4:召唤物太强 → 玩家失去主导感

**症状**:角色的召唤兽/宠物**自动击杀**敌人,玩家变成观众。

**业界案例**:
- 某些 MMO 召唤型职业(玩家放出召唤后泡茶看戏)
- 鬼泣 5 V 早期争议:玩家本体输出弱,但召唤兽太强压制玩家存在感

**避坑方法**:
- **召唤物必须靠玩家本体补刀**(鬼泣 V 修正:召唤兽打死敌人后 V 必须按键补刀)
- 召唤物提供"决策维度"而非"伤害爆发"
- 玩家可以临时直接操控召唤物(类猎天使 Demon Slave)

---

## 反模式 5:循环单调 → 重复疲劳

**症状**:战斗循环就是"特殊技 → 普攻 → 等 CD"机械重复,无变化。

**业界案例**:
- 某近战角色早期设计(初版):特殊技 → 普攻循环 → 等 CD —— 玩家社区给反馈"太单调"
- 部分 MMO 战斗节奏

**避坑方法**:
- 加"变化触发器"——某条件触发后下一招变化(类绝区零派生分支普攻)
- 加"资源叠层"——叠到 N 层后招式形态切换(类崩坏 3 / FF16 Stagger 倍率)
- 加"随机分支"——同一招有 2-3 种触发(类某些 boss 在某阶段有 A/B/C 概率分支招式)

---

## 反模式 6:全屏 AOE 无躲位 → 不公平

**症状**:Boss 释放全屏 AOE,玩家**没有任何安全区**,只能硬吃伤害或秒死。

**业界案例**:
- 一些设计糟糕的 raid boss(无脑全屏 + 高伤)
- 艾尔登兽被批评:hitbox 太大 + 自身位移 → 玩家追着打疲劳

**避坑方法**:
- 全屏 AOE 必须有"安全区"(Hades 2 Chronos 时钟数字光圈)
- 或者全屏 AOE 必须有"破解方式"(完美弹反 / 特定招式打断 / 跳到 boss 头上)
- 或者全屏 AOE 是"演出技"(伪秒杀但实际有 0.5 秒判定窗给玩家躲)

---

## 反模式 7:招式 telegraph 模糊 → 必须背板

**症状**:Boss 招式没有清晰的前摇提示,玩家**第一次见必死**,必须背板才能过。

**业界案例**:
- 早期魂系部分 boss 被诟病"随机大招"
- 某些"硬核"游戏(高难度但靠记忆而非反应)

**避坑方法**:
- 每招前摇有**视觉+音效双通道提示**
- 复杂招用"红光 / 黄光 / 蓝光"多色语义化(各项目应定义自己的视觉预警契约,见 [../conventions/TEMPLATE.md](../conventions/TEMPLATE.md))
- 招式动画时长 ≥ 0.5 秒,给玩家"看到 → 反应"窗口
- **新手第一次看到这招应该能猜对应对方式**

---

## 反模式 8:必须特定角色才能玩 → 卡平衡

**症状**:某关卡 / 某 boss 必须用特定角色 / 配队才能过,**玩家被强制抽卡**。

**业界案例**:
- 某些米哈游"必须主 C X"的深渊关卡(被吐槽"为了卖卡设计")
- 强制属性克制关卡(必须冰属性才行)

**避坑方法**:
- 关卡设计要支持**至少 3-5 种通关路径**
- 强属性克制可作为"高效路径"奖励,但不是"唯一路径"
- 测试关卡时用"非主流配队"试通关——能通 = 合格

---

## 反模式 9:无脑通关 → 没有挫败感 / 成就感

**症状**:游戏太简单,玩家"无脑按ABCD"即可通关,**没有任何挑战**。

**业界案例**:
- 部分 JRPG / 二游被吐槽"自动战斗就行"
- FF16 部分玩家批评"按 cooldown 释放就行"

**避坑方法**:
- 必须有"挫败期 → 突破期"曲线(参见 [设计审美](design-aesthetics.md) 难度曲线)
- 加分级难度(普通 / 困难 / 噩梦 / 地狱)
- 加评分系统(SSS / S / A 等)鼓励"打得更好"
- 加"挑战模式"——必通关后才解锁

---

## 反模式 10:机制堆砌 → 学不动

**症状**:角色 / Boss 同时拥有 5+ 个独立机制,玩家**记不住、学不动**。

**业界案例**:
- 部分二游"豪华套餐"角色(每个角色 7 个独立系统)被吐槽"看说明书都要 30 分钟"
- 某些 ACT 续作机制堆叠(忘了主线机制)

**避坑方法**:
- 单角色机制 ≤ 3 个"独立系统",其他都是"主系统的派生"
- 学习曲线**渐进式**:第一关只用机制 1,第二关引入机制 2...
- 教学关明确教每个机制
- 角色 UI 一眼能看出"3 个机制都在哪",不用翻菜单

---

## 反模式 11:无视觉记忆点 → 角色平庸

**症状**:角色设计完后玩家**记不住任何细节**,讲不出"独有"。

**业界案例**:
- 某些二游"中规中矩"的角色被吐槽"换皮"
- 没有专属梗 / 没有出圈招式

**避坑方法**:
- 每个角色 / Boss 至少 1 个"被玩家取外号"的招(元规则 14)
- 招式名要朗朗上口("水鸟乱舞" > "三段旋转斩攻击")
- 视觉特效要差异化(每个角色一个独有色系/视觉符号)
- **预测玩家会取的外号**——预测不出来就不够独特

---

## 反模式 12:战斗与剧情割裂 → 沉浸感断裂

**症状**:战斗时角色个性全无,剧情时又不打架——**玩法和叙事是两张皮**。

**业界案例**:
- 部分 JRPG"战斗 + 剧情切镜头"的传统模式
- 某些二游"剧情和角色玩法毫无关联"

**避坑方法**:
- 战神 2018+ 一镜到底(战斗与剧情无缝)
- 哈迪斯"死亡 = 叙事推进"(死回家有新对话)
- 角色的战斗机制要**对应性格**(沉稳的角色用蓄力,跳脱的角色用快连击)
- Boss 的招式要**对应剧情角色弧光**

---

## 反模式 13:养成强迫 / 抽卡压力

> ⚠️ 本条主要针对**抽卡 / gacha + 在线运营**品类;买断 / 单机项目按"0 解锁即完整体验"理解即可,无 PVP 的项目跳过 PVP 条目。

**症状**:必须氪金 / 时长养成才能解锁核心玩法,**新角色付费墙太高**。

**业界案例**:
- 部分二游"必须满命座才好玩"
- 强制氪金的"PVP 平衡"

**避坑方法**:
- (抽卡品类)0 命座 / 0 叠影也要有完整玩法,养成是"锦上添花"不是"必备";买断制对应"不点技能树也能通关"
- (若有 PVP)PVP 数值差距不超过 ~30%
- 养成线路解耦(类 FF7 Rebirth 武器招式继承,用够该武器后招永久解锁)

---

## 反模式 14:UI 信息过载

**症状**:战斗时屏幕上 10+ 个数字 / Buff icon / 提示符,**玩家看不过来**。

**业界案例**:
- 部分 MMO 战斗 UI 复杂
- 某些 ACT 添加多机制后 UI 失控

**避坑方法**:
- 主战斗 UI ≤ 5 个核心元素(示例:HP / 资源条 / 架势条 / 形态 / 大招——按你项目实际有的要素)
- 次要信息折叠(Buff 列表悬浮显示)
- 关键提示用画面边缘醒目色闪烁,不堆中央

---

## 反模式 15:无成长闭环

**症状**:打通 Boss 后**什么都没获得**——招式 / 装备 / 玩法都没解锁。

**业界案例**:
- 部分线性叙事 ACT(打过就过,没记忆)
- 早期魂系部分 boss 战利品平庸

**避坑方法**:
- 学黑神话:打过 boss 即可学其招式(精魄变身)
- 学 MH:讨伐 = 解锁该召唤的玩家可用版
- 学对马:完成支线学到独家"传说技"
- 至少每个 boss 解锁 1 个新玩法(团队联动技变体 / 武器 / 形态)

---

## 反模式 16:玩家失败时的挫败感不能转化

**症状**:玩家死了之后**没有"再来一次"动机**——只是骂街。

**业界案例**:
- 某些"无解必杀"招(玩家觉得"运气不好")
- 某些惩罚过重的死亡设计(类似传奇游戏的掉装备)

**避坑方法**:
- 玩家失败后必须能复盘"我哪里错了"(可读性是前提)
- 死亡 = 叙事推进(哈迪斯)or 死亡 = 学到新东西
- 复活成本不能太高(类只狼回升机制)
- 失败提示要"建设性"——告诉玩家可以试什么解法

---

## 设计反模式自检表

设计完每个角色/Boss 后过一遍:

```
[ ] 没有单一机制收益过高
[ ] 蓄力期间玩家有操作
[ ] 大招 < 5 秒或有可操作空间
[ ] 召唤物需要玩家本体配合
[ ] 战斗循环有变化(不是机械重复)
[ ] 全屏 AOE 有安全区/破解方式
[ ] 招式 telegraph 清晰(新手能猜对应对方式)
[ ] 关卡支持多种通关路径(不卡角色)
[ ] 有挫败 → 突破曲线
[ ] 单角色机制 ≤ 3 个独立系统
[ ] 至少 1 个"被玩家取外号"招
[ ] 战斗 + 剧情有关联
[ ] (抽卡品类)0 命 / 0 解锁就完整
[ ] UI 主要素 ≤ 5 个
[ ] 通关有奖励(招式/装备/玩法)
[ ] 失败可复盘
```

打钩 < 12/16 → 重新审视设计。
