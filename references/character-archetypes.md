# 角色核心机制原型库(V2)

ACT / ARPG 角色机制设计的可复用骨架。综合业界 25+ 款 ARPG 调研 + 多个原创角色实战沉淀的 **12 种核心机制原型**——设计新角色时**选其一改造 / 组合多种 / 或按 [元规则](meta-rules.md) "机制创新法则"从灵感种子推导新原型**。

⚠️ **核心方法论:不要从原型挑选填空,要从灵感种子推导,然后看落到哪个原型**。原型只是"已知样本",不是"必须选其一"。

> 下面每个原型头部不再标注固定的"分类"——角色分类轴(职业 / 定位 / 稀有度 / 武器型等)由你的项目自定义,详见 [项目约定模板](../conventions/TEMPLATE.md)。资产命名(`pre` / `charge` / `release` / `recover` 等)也只是**示例命名规范**,定义你自己的即可。

## 基础 5 种原型

### 原型 A:长按蓄力削韧 + 自 Buff

**核心套路**:
1. 特殊技长按蓄力,期间防御 Pose + 自身 Buff(韧性↑ / 防御↑)
2. 蓄力越久越强(中削韧 vs 高削韧)
3. 进阶扩展:把蓄力期间承伤转化为后续输出
4. 被动:HP 低时触发霸体+减伤

**资产关键**(示例命名):前摇一段 + 蓄力 loop 一段 + 释放(中削韧)一段 + 强化释放(高削韧)一段 + 恢复一段

**适用方向**:护甲职业、武术系、击破型副C

**进阶改造灵感**:
- 学 Hades 2 Ω 蓄力变招(所有招式都有普通版/蓄力版)
- 学黑神话识破回棍势能量(蓄力期间被击 → 触发反弹判定 → 不只是被动 Buff)

---

### 原型 B:标记 + 变形普攻循环

**核心套路**:
1. 特殊技标记敌人 → 命中数量越多自拐越高
2. 普攻形式变化(失控形态,AOE 多 hit)
3. 命中标记敌人额外回能
4. 拉怪(标记的无霸体怪被拉近)

**适用方向**:主 C 输出、AOE 群伤、平 A 流大 C

**进阶改造灵感**:
- 标记叠层引爆(1 层 +AOE / 3 层 +多 hit / 5 层 +引爆)
- 学绝区零属性紊乱(命中已上某属性异常的敌人 → 450% 加成 + 提前结算)

---

### 原型 C:增益辅助 + 资源强化

**核心套路**:
1. 入场即特殊技,持续性 loop 形态(如龙卷风)
2. 给登场角色 buff
3. 资源攒满替换强化版特殊技(百裂拳→重拳)

**适用方向**:辅助、增益副 C、上场即放

**进阶改造灵感**:
- 学 Hades Boon 染色(给队友每个招式类型分别附加染色——普攻染雷 / 特殊技染冰)
- 学 FF7 Rebirth Tifa 模式(增益 = Stagger 倍率工程师,把全队倍率从 160% 推到 230%+)

---

### 原型 D:远程控距 + 长按持续

**核心套路**:
1. 武器轮换(单/双枪→机关枪→大炮)
2. 长按持续射击+移动(双摇杆瞄准)
3. 控距打法(同攻击不同距离不同伤害)
4. 击飞/浮空衔接持续浮空(自循环)

**适用方向**:远程输出、控距流、双摇杆瞄准操作

**进阶改造灵感**:
- 学 MH 弩枪(蓄力 → 弹种切换 → 装填管理)
- 学剑魔 Q(距离判定 = 不同伤害,鼓励"走位补刀")

---

### 原型 E:双形态切换

**核心套路**:
1. 双形态:常态(灵活/控弹/Buff) + 重装态(肉身/霸体/盾击)
2. 绝技切换形态(如召唤机甲砸落)
3. 闪避接特殊技触发超载冲击(全击飞包括 Boss)
4. 形态间能量消耗(重装态每秒耗能,耗完自动退)

**适用方向**:主推、复杂操作、双形态、高上手难度

**进阶改造灵感**:
- 学 FF16 Eikon 30 秒一切换(从双形态升级到 3+ 形态轮转)
- 学忍龙武器切换实时性(mid-combo 切换不打断 combo 计数)

## 原型组合搭配示例

- **原型 A + 原型 H**:武术家蓄力期间给队友 Buff,且能拉 Stagger 倍率
- **原型 B + 原型 D**:远程枪手,标记敌人后远程子弹追加伤害
- **原型 E + 原型 G**:双形态各自有 Eikon 槽位(常态挂 3 个 Eikon / 重装态挂 3 个 Eikon)
- **原型 F + 原型 K**:武术三态,每态对应一个"博弈规则修改"(劈态破防 / 立态减少敌人霸体 / 戳态延长玩家弹反窗口)
- **原型 I + 原型 J**:召唤兽 + 武器槽组合(召唤多个不同武器特化的小弟)

## 设计新角色时的起手流程

1. **从灵感种子推导**(见 [元规则](meta-rules.md) "机制创新法则")
2. **不要直接从本文件挑原型**——先推导再看落到哪个原型
3. 如果推导出**全新原型**(超出本文件 12 种),沉淀进本文件作为原型 R / S...
4. 选定原型基线后:Read 该原型代表角色的战斗设计文档做参考样本
5. 对照 [元规则](meta-rules.md) 做强度评估(对照你的强度评估清单,命中条目偏少就改设计)
