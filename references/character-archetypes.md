# 角色核心机制原型库(V2)

ACT / ARPG 角色机制设计的可复用骨架。综合业界 25+ 款 ARPG 调研抽象的 **12 种核心机制原型(业界 F–Q 型)**——设计新角色时**选其一改造 / 组合多种 / 或按 [元规则](meta-rules.md) "机制创新法则"从灵感种子推导新原型**。

⚠️ **核心方法论:不要从原型挑选填空,要从灵感种子推导,然后看落到哪个原型**。原型只是"已知样本",不是"必须选其一"。

> 下面每个原型头部不再标注固定的"分类"——角色分类轴(职业 / 定位 / 稀有度 / 武器型等)由你的项目自定义,详见 [项目约定模板](../conventions/TEMPLATE.md)。资产命名(`pre` / `charge` / `release` / `recover` 等)也只是**示例命名规范**,定义你自己的即可。

## 业界调研 12 种原型(F–Q 型)

业界 25+ 款 ARPG 调研抽象出的 12 种机制原型,完整描述见 [角色原型扩展](character-archetypes-extended.md):
- F 型 多态共享资源池(黑神话 / Yakuza)
- G 型 形态身份切换(FF16 Eikon)
- H 型 Pressure 工程师(Tifa 模式)
- I 型 召唤指挥(鬼泣 V / FF7 Cait Sith)
- J 型 武器组合槽(猎天使 / Aspect)
- K 型 博弈规则修改型(永劫 / DD2 Trickster)
- L 型 远程姿态独立型(MH 弓弩)
- M 型 变形武器双形态(BB Trick Weapon)
- N 型 工具箱猎人型(Witcher 5 印记)
- O 型 武器组装型(Lies of P 刃+柄)
- P 型 工具箱辅助槽(Halo Equipment)
- Q 型 后台战斗型(Trigger / Memosprite)

## 原型组合搭配示例

- **原型 F + 原型 K**:多态共享资源池 + 博弈规则修改(武术三态,每态改一条规则:劈态破防 / 立态减敌霸体 / 戳态延长玩家弹反窗)
- **原型 I + 原型 J**:召唤指挥 + 武器组合槽(召唤多个不同武器特化的小弟)
- **原型 G + 原型 H**:形态身份切换 + Pressure 工程师(切到专属形态时化身倍率引擎,把全队 Stagger 从 160% 推到 230%+)
- **原型 L + 原型 N**:远程姿态独立 + 工具箱猎人(远程架势下切换多种印记 / 陷阱 / 涂油,控场收割流)
- **原型 M + 原型 O**:变形武器双形态 + 武器组装(刃 / 柄分别变形,组合出不同招式树)

## 设计新角色时的起手流程

1. **从灵感种子推导**(见 [元规则](meta-rules.md) "机制创新法则")
2. **不要直接从本文件挑原型**——先推导再看落到哪个原型
3. 如果推导出**全新原型**(超出本库 12 种),沉淀进本文件作为原型 R / S / T…(或放你本地的项目知识区)
4. 选定原型基线后:Read 该原型代表角色的战斗设计文档做参考样本
5. 对照 [元规则](meta-rules.md) 做强度评估(对照你的强度评估清单,命中条目偏少就改设计)
