# 配队协同设计专题(V2)

ACT / ARPG 多角色协同与配队设计的方法论。基于业界 50+ 款游戏调研(含神配队解构 / Roguelite Build / 反派群组 / 平衡哲学 四维深挖)。

⚠️ **核心定位**:本文件专门处理"多角色协同 / 配队互动"问题,跟 [角色原型库](character-archetypes.md)(单角色设计)、[Boss 范式库](boss-paradigms.md)(boss 设计)、[元规则](meta-rules.md)(元规则)互补。

## 一、25 条配队设计元规则

### A. 团队结构与角色定位

#### 1. **一支神队 = 一个 win condition + 三角色围绕这个 condition**

国家队 win condition = 班尼特 atk buff + 蒸发反应;**行秋/香菱/雷神都围绕"如何更高效触发蒸发"**。**不要做"功能堆叠"队伍**(4 个治疗 / 4 个输出),每个角色对应 win condition 的不同 phase。

#### 2. **Snapshot 机制 = 神队的隐藏发动机**

行秋/香菱因 **snapshot 班尼特 buff** 而成为可被 stack 的"持续输出"——元素爆发瞬间快照 buff,**之后哪怕班尼特换下场,buff 整 10 秒都吃满**。这是配队中**最容易被低估的机制层**。

**设计要点**:每个 buff 必须明确是 **snapshot(快照)还是 dynamic(动态)**。米哈游后来意识到这是漏洞,把许多 buff 改成 dynamic,但国家队已经诞生。

#### 3. **反向 buff 链**:让"reverse 玩家"成为流派

胡桃配钟离不配班尼特——因为胡桃 50% 血量以下伤害最高,班尼特治疗会"踢出半血血祭"。钟离护盾保命但不治疗,正好对接胡桃低血玩法。**创造"看似有害实际有利"的状态**(低血、debuff、自伤)让支援出现 niche。

#### 4. **"Driver + Sub-DPS + Support × 2" 是 3D ACT 最稳模板**

- **Driver**:负责场上输入(普攻、移动)
- **Sub-DPS**:场下持续输出(脱手型)
- **Support × 2**:buffer + healer/shielder

**变种**:1 主 C + 2 副 C + 1 增益(FF7 Rebirth)或 3 主 C 轮换(PGR 灼霜队)

#### 5. **派系/羁绊系统**能解决"故事 + 数值"双重困境,但隐含 powercreep 风险

ZZZ Cunning Hares 初心好,但 S2 派系扩张后旧派系被冷落。**派系绑定要保留跨派系搭配可能**(同派系 2 件加 buff,但不锁队伍)。

---

### B. Synergy 时序

#### 6. **Combo 设计 = "T-1 → T+0 → T+N" 三段节奏**

- **T-1**:buffer 上场撒 buff(行秋 E、夜兰 Q)
- **T+0**:主 C 上场打输出(胡桃 E)
- **T+N**:长尾持续效果(旋火轮 10 秒持续)

**三段缺一不可**:缺 T-1 主 C 没 buff,缺 T+N 输出窗口太短。

#### 7. **触发条件分层:免费基础协同 + 资源消耗高阶协同**

| 样本 | 免费层 | 昂贵层 |
|---|---|---|
| FF7 Rebirth | Synergy Skill(长按 Guard + 面键) | Synergy Ability(双 ATB) |
| DBFZ | Z-Assist(6 秒 CD) | Sparking Blast(全局 1 次) |
| 绝区零 | Quick Assist(免费救场) | Chain Attack(Daze 槽满) |

#### 8. **"脱手"是 3D ACT 配队的灵魂**

班尼特/行秋/夜兰/纳西妲都是脱手强角色——**"脱手"让玩家只用 1 个角色操作但 4 个角色都在输出**,解决了 ACT 的"一次只能操作一个"瓶颈。这是 ACT 区别于回合制的设计层。

### C. Roguelite Build 设计

#### 9. **Build 多样性来自"流派种子"而非"每个角色都强"**

- Hades 9 god × 4 boon slot = 36 维度
- 杀戮尖塔 Ironclad 的 Strength / Block / Bloodletting 是预设流派种子
- **设计师预设 5-10 个流派种子**,让玩家在 100+ build 里识别归属

#### 10. **必抽 ≠ 强 build。强 build 应该 emergent 而非 prescriptive**

Hades Duo Boon 强但**不可控**(玩家无法直接选,只能通过 Keepsake 间接提概率)。**如果某 build 必须靠某稀有道具触发,那个道具的稀有度应该 ≤ Epic/Rare**。

#### 11. **Aspect / Mutation / Trait 系统让"同一基础道具 4 种用法"= 内容量翻 4 倍**

- Hades 24 个 Aspect 让 6 把武器变成 24 个 build 起点
- Returnal Trait 让一把枪有数十种组合
- Vampire Survivors 武器 evolution + union(两把基础武器 = 一把顶级武器)

#### 12. **物品堆叠的"linear / multiplicative / hyperbolic"三模型 是 build 复利的核心**

- **Linear**(Soldier's Syringe 攻速):N 次 = 效果 × N
- **Multiplicative**(Crowbar):(1+0.4)^N
- **Hyperbolic**(Tougher Times 减伤):每件递减边际收益

**不要全 linear**(无脑堆) / **不要全 hyperbolic**(玩家放弃堆) — 三类混合制造 build 张力。

### D. 反派 / Boss 群组

#### 13. **多 boss 协同 = 优先级目标选择(priority targeting)的训练场**

O&S(黑魂)、Mantis Lords、Theseus + Asterius 都让玩家做"先杀谁"的策略决定。**多 boss 不应该"一锅炖"——必须有明确的"优先级 trade-off"**。

#### 14. **多 boss 设计三铁律** 🆕

1. **攻击模式必须区分明显**(近 vs 远 / 快 vs 慢 / 高频 vs 低频)
2. **视野上必须能同时显示**(避免"看不见的攻击"打击体验)
3. **剩 1 个时必须升级**(O&S 经典:吸收能量满血加强)

#### 15. **1 超强 boss vs 1+1 boss 选择 = 深度优先 vs 广度优先**

- **深度优先**(FromSoft 苇名一心 / 艾尔登梅赛尔):学单 boss moveset
- **广度优先**(Hades / Hollow Knight Mantis):学多 AI 协同处理
- **叙事接力**(黑神话):非协同,但通过"地缘 + 派系叙事"形成连战感

#### 16. **玩家"分而治之"工具决定多 boss 体验**

蒸发 / 草元素 / 雷反应等 AOE 反应让玩家"打一个伤多个" / 锁定系统 + camera 配置直接决定能否分而治之 / 鬼泣 5 三主角分线 = "用叙事承担多 boss 量"的优雅解。

### E. 平衡 / Powercreep

#### 17. **"环境驱动 powercreep" > "角色驱动 powercreep"** 🆕

- 推**新元素 / 新机制**(Dendro、Fontaine arkhe)= 让老角色复活
- **直接调数值** = 玩家氪金被打脸
- 环境驱动需要长期规划,但回报是 4+ 年生命周期

#### 18. **每个角色必须有"至少一种功能不被取代"** 🆕

- 班尼特:火附着 + 治疗 + 攻击力 buff 三合一
- 万叶:风扩散 + 元素伤害加成 + 抗性减
- 钟离:护盾 + 抗性减 + interrupt resistance

**没有"功能护城河"的角色注定被 powercreep**。

#### 19. **永久 debuff + 自我增益 + 低风险触发 = 必禁** 🆕

DBFZ Lab Coat 21 反例:Photon Pulse 大招 = 21% 永久减伤 + 21% 自伤增益 + 21 帧指令投 → 直接破坏伤害公平假设 → CEO 2022 被禁。**这类角色构造"全民必带"反 meta 锁**。

#### 20. **Patch 五工具箱按优先级使用**

| 优先级 | 工具 | 用法 |
|---|---|---|
| ⭐⭐⭐ | 环境改变 | 新元素 / 新机制 / 新敌人类型 |
| ⭐⭐ | 出新角色让旧角色复活 | Furina 让胡桃焕新 |
| ⭐⭐ | buff 老角色 | ZZZ 老角色免费加强 |
| ⚠️ | 机制改动(只对新角色生效) | 米哈游 snapshot 改 dynamic 不追溯 |
| 🚫 | 数值微调 | 玩家会反弹"我抽的角色被砍了" |

#### 21. **新 C + 新支援"门当户对"绑定** 🆕

2024-2026 二游标准做法:**每个版本主 C + 支援同期推出**,新 C 自带一套适配阵容。

**样本**:
- 崩铁:Aglaea + Sunday / 流萤 + Ruan Mei
- 鸣潮:Phoebe + Brant / Cantarella + Lupa
- 绝区零:Hugo + Trigger
- GF2:索米 + 后续冰主 C

**设计哲学**:避免"无支援的新 C 滞销" + 强迫老玩家持续投入抽支援。**老角色靠"接驳新支援"保持流通**(Acheron 至今 S+ 靠 Sunday/Cyrene 提速)。

#### 22. **共享资源池作为队伍级决策** 🆕

队伍共用一根资源条 → "谁攒、谁花、什么时机花"成为配队核心策略。

**样本**:
- **终末地全队 SP 共享**(300 上限 3 段)
- **鸣潮协奏能量**
- **苍蓝档案 EX cost 共享池**(每秒 +1,谁先用谁规划)
- **GF2 导染系统**(100→200 全局 buff 表)

**设计要点**:**资源管理上升到队伍级**,不是每个角色独立蓄力。这能让"配队"在战斗中持续产生决策。

#### 23. **承诺老角色加强 = 二游长线信任** 🆕

公开告知玩家"老角色不会下线" 是 2024-2026 二游对抗 powercreep 焦虑的关键招式。

**样本**:
- 米哈游 2024 流水下滑后**公开承诺**"老角色本体加强已经提上日程"
- 鸣潮 1.x 起持续给老角色补送 + 终响版本调整
- ZZZ "老角色免费加强"

**设计要点**:数值加强可以无声,但"承诺"必须**公开**。这是平衡哲学之外的"沟通哲学"。

#### 24. **端游 / 深境塔 / Raid 分摊职能定位** 🆕

不同端游模式让"输出/击破/增益"在不同模式各有最优解。

**样本**:
- **崩铁三模式**:MoC(单 boss 持续输出)/ PF(群攻清杂)/ AS(机制理解 + 少行动数)
- **鸣潮 ToA / Whimpering Wastes / Endstate Matrix**:多模式 + 多区
- **苍蓝档案总力战 / 大决战 / 联合作战演习**

**设计要点**:**避免"一队打天下"**,让每种定位都有"主场"。这是反"必带角色"的环境层设计。

#### 25. **演出"短化 + 仪式化"双轨** 🆕

战斗演出整体仍华丽,但**节奏要分级**:
- **大招演出"短促化 + 可跳过"**(玩家社区疲劳信号)
- **招牌角色出场 / 大招"仪式感拉满"**(崩铁 3.0 每个新 5 星专属过场)

**演出预算从"全角色平均"转向"招牌角色重点投资"**。

## 三、配队设计的通用方法论(适合操作简单 / 大众向的队伍系统)

下面这组方法论适用于"想降低配队理解门槛、但仍保留战前搭配策略"的项目。术语保持项目无关:把"团队联动技 / 切人技"理解为任何"角色间的主动协同动作",把"角色离场后的延续行为 / 召唤体"理解为任何"角色切走后仍在场产生效果的机制"。

### 1. 先定核心出伤点(win condition),辅助围绕它服务

设计一支队伍,**第一步永远是先回答"这支队伍主要靠什么打伤害"**,再设计辅助如何服务它。常见的核心出伤点类型:

| 类型 | 说明 |
|---|---|
| 单技能高倍率 | 某个重击 / 特殊技 / 终结技承担主要伤害 |
| 状态 / 层数结算 | 辅助先生产层数或状态,主输出读取后爆发 |
| 高倍率技能变种 | 开某状态后,某类技能性质改变(如普攻变重击逻辑) |
| 双核 / 多核循环 | 两名或多名角色都承担关键伤害与循环价值 |
| 队伍机制伤害 | 伤害来自队伍公共机制而非单一技能 |
| 防反 / 破绽爆发 | 靠闪避反击、格挡、破绽读取打高收益 |

普通主推队伍优先用"单技能高倍率""高倍率技能变种"等**简单清晰**结构;多核循环、强对轴队伍不作为大众默认体验。

### 2. 辅助三层模型:帮什么内容 / 怎么承载 / 谁来结算

辅助设计拆成三个正交的层,分开想会清晰很多:

```
辅助内容 = 帮什么(增伤 / 状态生产 / 机制改写 / 回能 / 生存 / 控场 / 副输出)
承载方式 = 怎么帮(入场即时 / 长时铺场 / 离场延续 / 接力预存 / 常驻被动 / 命中触发)
结算关系 = 谁来用(谁读取这个状态、谁吃这个 buff、谁完成爆发)
```

关键洞察:**"长时铺场"不是一种独立的数值内容,而是一种承载方式**。例如"生成 20 秒领域,领域内重击增伤"——内容是重击增伤,承载方式才是长时铺场。同理**"返场读取"不是一种独立辅助内容,而是一种结算关系**。把这三层分开,能避免把"怎么帮"和"帮什么"混为一谈。

### 3. 时间宽容度原则:buff 持续长一点、条件少一点

面向大众玩家的协同设计,效果应尽量**宽容**:

```
持续长一点
条件少一点
反馈明显一点
不大量依赖短时精确对轴
```

**优先**:长时铺场 / 状态生产 / 返场读取 / 生存回转 / 角色离场后的延续行为。
**慎用**:持续 5 秒以内、必须主输出立刻接管、错 1-2 秒就大幅亏损的短轴 buff。

校验方法:**晚 2-3 秒操作是否仍然有效?漏掉一次联动是否还能继续打?** 答案应是肯定的。

### 4. 配队设计流程骨架

```
1. 定队伍结构(确定核心 + 辅助的数量与分工)
2. 定核心出伤点(先回答"靠什么打伤害")
3. 定辅助内容(每名辅助分别帮核心做什么)
4. 定承载方式(入场即时 / 长时铺场 / 离场延续 / 接力预存)
5. 设计联动链(谁起手、谁触发、谁接管、谁延续、谁结算)
6. 校验时间宽容度(晚 2-3 秒是否仍有效)
7. 做视觉正反馈(资源就绪、状态命中、关键结算都要让玩家看懂)
```

视觉反馈要点:正确搭配时让玩家**直观感到**"这队很顺"——资源条亮得更快、有粒子从攻击目标飞向队友、关键结算有强顿帧 / 专属音效 / 数字样式变化。玩家不需要懂公式,只要感受到"我配对了、我打对了、这一下明显更爽"。

### 5. 要点控制风险表

| 风险 | 控制方式 |
|---|---|
| 配队理解成本过高 | 主推一个清晰的默认队伍结构,把复杂结构留给进阶/特色队 |
| 过度精确对轴 | 联动 buff 做长时铺场 + 返场读取,减少短轴依赖 |
| 辅助变成"万能必带" | 辅助要有"擅长子领域" + 加反向限制条件,不做全队通用最优 |
| 玩家感知不到配队收益 | 用 UI 效果、特效、数字、音效强化反馈 |
| 角色离场延续行为变成"切谁都乱打一套" | 延续行为必须有明确服务对象 + 可见正反馈,动作固定、时间可控 |
| 可选状态轴过早复杂化 | 若项目有额外的状态轴工具箱(异常 / 削韧 / 标记等),前期少用,作为后续扩展 |

## 四、设计新配队 / 联动技的起手流程

1. **先想"为啥这俩要配"**——不是数值,是机制独家性 + 剧情情感
2. **画 buff 链路 T-1/T+0/T+N 三段时序**
3. **决定触发分层**(免费层 + 昂贵层)
4. **决定资源经济学**(独立 / 共享 / 双层)
5. **决定动画时长**(0.5-1s 频繁 / 3-5s 重要 / 5-8s 终极)
6. **测试涌现性**——这套机制是否支持"非预想"的玩家组合?
7. **明确每个 buff 是 snapshot vs dynamic**(避免后续设计困境)
8. **对照 25 条元规则**逐条评估
9. **三大警示自检**(见下)
10. **平衡哲学预案**:这角色 4 年后会不会被 powercreep?有没有"功能护城河"?

### 三大配队设计警示

#### 警示 1:**"国民老婆"陷阱**
辅助 buff 太通用 → 几乎所有队伍必带 → 设计师未来再设计辅助都被锁死。**避坑**:辅助每个有"擅长子领域",加"反向限制条件"。

#### 警示 2:**"强制必带"陷阱**
A 必带 B → 玩家被逼抽。**避坑**:强联动必须有剧情/羁绊支撑,让玩家"想用"而非"被逼用"。

#### 警示 3:**"4 个独狼"陷阱**
协同设计太弱 → 4 人合作 = 4 个单刷叠加。**避坑**:保证 1+1 > 2(独立机制效果)+ 涌现性强(50/200 倍可能性)。

## 五、核心总结

**25 条元规则**,覆盖团队结构 / Synergy 时序 / Roguelite Build / 反派群组 / 平衡哲学 五大模块。

**最重要的元设计哲学**:

> **"涌现性(emergence)"超过"规定性(prescription)"** —— 最强的配队、最厚的 build 树、最深的 boss 战、最长寿的平衡,都是设计师**"预设可能性空间"**而非**"指定标准答案"**的产物。

这是从 Genshin 国家队、Hades Duo Boon、O&S 双 boss、Bennett 4 年不下线背后的共通设计哲学。
