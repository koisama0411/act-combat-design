# 数值与养成规范(工程化基线)

⚠️ **本文件性质**:与 [打击感技术](hit-feel-tech.md) 同性质——**工程化技术规范**,可直接作为项目"数值公式 / 击破阈值 / 养成设计"配置基线。

业界 14+ 款 ACT/ARPG 调研沉淀。**核心心法**:现代二游主流 = "**纯乘法 4 bucket + 选择性 snapshot + 击破赛道独立词条**" —— 5 年验证最稳架构。

---

## 一、伤害公式 5 大架构哲学

| 架构 | 代表 | 优点 | 缺点 | 工程化建议 |
|---|---|---|---|---|
| **纯加法** | 早期 RPG / 端游回合 | 易理解、易调平衡 | 词条堆叠失策略性 | ❌ 不推荐 |
| **纯乘法 4 bucket** | 原神/星铁/鸣潮/ZZZ | 短板补长策略性 | 易膨胀 | ⭐ **主流推荐** |
| **复合(乘+加)** | MH Wilds / 巫师 3 | 多层次 | 调平难度大 | 装备品质感强的场景 |
| **Snapshot(瞬时锁定)** | Genshin 部分技能 | 后台 buffer 玩法 | 玩家不易理解 | 召唤物/持续技能用 |
| **Dynamic(实时重算)** | 大部分普攻 | 直观 | 后台无效 | 默认选项 |

**结论**:**纯乘法 4 bucket + 选择性 snapshot**(召唤物/持续技用 snapshot,普攻用 dynamic)。

## 二、6 款主流游戏伤害公式

### 1. 原神(纯乘法 4 bucket + Snapshot/Dynamic 双轨)

```
最终伤害 = 基础伤害(技能倍率×攻击/HP/防御)
        × (1 + 总伤害加成%)
        × 暴击修正(1 + CR × CD)
        × 防御修正
        × 抗性修正(分段:<0/0-75%/≥75%)
        × 元素反应倍率
```

**关键参数**:
- 元素反应:蒸发 2.0×(水触发火)/ 1.5×(火触发水);融化同;超载固定 4.0×;超绽放 3.0×
- EM 软上限:`(2.78 × EM) / (EM + 1400)`,300 EM 是性价比拐点
- 抗性公式:RES<0 → (1-RES/2);0-75% → (1-RES);≥75% → 1/(4×RES+1) 软衰减
- 双爆词条比 **CR:CD = 1:2**

**Snapshot vs Dynamic**:
- **Snapshot**:释放瞬间锁定属性(行秋水剑、香菱旋火轮、Albedo 阳华)→ 后台 buffer 玩法基础
- **Dynamic**:每帧重算(神里 Q、八重 E、普攻)→ buff giver 必须前台

### 2. 崩铁(乘法 + 韧性折扣双层)

```
普通伤害 = 基础DMG × 韧性折扣(0.9未破/1.0已破) × CRIT × DMG加成 × DEF修正 × RES修正

韧性削减 = (基础减韧 + 加性减韧)
        × (1 + 减韧增加%)
        × (1 + 击破效率% + 韧性易伤%)  -- 击破效率 cap 300%
        × 招式倍率
```

**Break DMG 特性**:不能暴击、不吃 DMG% 加成,**击破特攻是独立词条赛道**

**Super Break (2.2 引入)**:每打 broken 敌人都触发,线性堆击破特攻无软上限

### 3. 鸣潮(乘法 + Tune Break 独立赛道)

```
最终伤害 = 基础伤害 × (1 + 元素DMG% + 攻击型DMG%) × CRIT × DEF × RES × VULN
```

**Tune Break**:不吃 ATK/CRIT,只看角色 Tune Break Boost + 敌人 Class —— 把击破伤害**完全剥离主 DPS 赛道**

### 4. 绝区零(双赛道:Standard + Anomaly)

```
Standard DMG = (ATK × Skill Mult) × (1 + DMG Bonus) × CRIT × DEF × RES × Stunned × VULN

Anomaly DMG = Anomaly Mult × ATK × (Proficiency/100) × Level Mult × DMG Bonus × Stunned

Daze 累积 = Impact × DazeMult × (1 + Daze增 - Daze减) × (1 - DazeRES) × (1 + DazeTaken)
```

**Stunned 倍率**:+50%~150%(因敌而异)+ 触发 Chain Attack

### 5. MH Wilds(物理 motion value + 锋利度 + 肉质)

```
最终伤害 = (True Raw × Motion Value × Sharpness Mod × Critical Mod) × Wound Mod × HZV / 100

True Raw = 显示攻击 ÷ 武器 Bloat(大剑/大锤 4.8 / 太刀双刀 3.3)
有效血量 = HP × (1 + DEF / 80)
```

**关键参数**:
- 锋利度倍率:红 0.5 / 橙 0.75 / 黄 1.0 / 绿 1.05 / 蓝 1.2 / 白 1.32 / 紫 1.39
- Critical Mod:默认 1.25;Critical Boost Lv3 → 1.40
- 弱点(HZV 45+):激活 Weakness Exploit
- Wound +20%(伤口未破时叠加 HZV)
- **状态异常阈值递增**:100 → 130 → 170(防"无限麻痹链")

### 6. 法环(6 系伤害 + scaling soft cap)

```
Total AR = Base DMG + (Base DMG × Scaling Mult × Stat Saturation)
最终伤害 ≈ AR × Motion Value × Buff × 部位修正 × 防御抵消 - 防御flat
```

**Scaling soft cap**:1-20 强 / 20-55 强 / 55-80 急衰减 / 80-99 几乎无收益。双手 → 力量 ×1.5

**Poise breakpoints**(PvE):
- 51 Poise:抗一段轻型 R1
- 101 Poise:抗一段中型 R1
- 129 Poise:抗一段大型 R1

## 三、击破系统 6 大要素 + 通用公式

### 通用积累公式

```
击破累积 = 基础值 × 招式系数(每招独立 daze mult)
        × 攻击方 Break/Impact 属性
        × 弱点/部位修正
        × 状态加成(已 broken 不再积)
        × 抗性修正(目标 Break RES)
```

### 6 大要素配置基线

| 要素 | 设计要点 | 数值参考 |
|---|---|---|
| **阈值** | 基于级别/类型/部位 | 小怪 30-60 / 精英 80-120 / Boss 150-250 |
| **积累公式** | 独立词条赛道(Break Effect / Impact)与 ATK/CRIT 独立 | 击破效率 cap 300%(星铁) |
| **窗口时长** | 普通失衡 5-10 秒,Boss stagger 10-15 秒 | FF7R 10-15s |
| **倍率** | 起点 +50%-60%,可堆 +150-200% | FF7R 160%→300%,ZZZ +50%~150%,法环 ×3.0 |
| **收益类型** | 至少 2 项叠加(倍率 + 行动延迟 / 处决 / 阶段切换) | 单收益感不够 |
| **UI 显示** | 三档:透明(显示条) / 半透明(视觉提示) / 隐藏(Focus Mode) | 二游走透明,硬核 ARPG 走隐藏 |

### 7 款击破系统对比

| 游戏 | 击破条 | 显示 | 累积 | 收益 | 暴击? |
|---|---|---|---|---|---|
| **只狼** | Posture | 接近破才显示 | 招架+攻击 | 处决式一击杀 | N/A |
| **法环** | Poise | 不可见 | 重武器/战技 | ×3.0 + critical hit | N/A |
| **ZZZ** | Daze | HP 下方独立条 | Impact × Daze Mult | +50-150% + Chain Attack | 是 |
| **星铁** | Toughness | HP 上方 | 元素技/弱点削减 | Break DMG + 行动延迟 + 0.9→1.0 | 否 |
| **战双** | Groggy/Stun | UI 不显数值 | 信号球 + 大招 | 增伤 + QTE 处决 | 是 |
| **MH Wilds** | Wound + Stun | Wound 视觉化(伤口) | 部位累积 + Focus Strike | Focus Strike 海量+强制倒地 | 否 |
| **FF7R** | Pressure→Stagger | 紫条/橙条 + 文字标记 | 弱点 + Pressured/Stagger Rate | 160% 默认,可堆 300% | 是 |

## 四、buff 叠加 4 大模式 + 防"国民老婆"4 铁律

### 4 大模式

| 模式 | 规则 | 例子 |
|---|---|---|
| **同名覆盖(高吞低)** | 同 buff 取最高 | Bennett 双人不叠加 / 星铁 Bronya 攻击 buff |
| **同名加性叠加(stack cap)** | 同 buff 加性叠加,有上限 | ZZZ Anomaly stacks / 鸣潮 Concerto |
| **不同名乘性叠加(独立 bucket)** | 同 bucket 内加性,跨 bucket 乘性 | 原神 ATK%(圣遗物+鸣神套+Bennett) |
| **稀缺独立乘子** | 极少数 buff 独立乘性绕开所有 bucket | Furina Q / Robin / Petri's Philter — **国民老婆强度阈值就在这里** |

### 4 铁律(防国民老婆)

1. **通用 ATK 加成不要给独立 bucket** —— 否则成 Bennett(永远不可替代)
2. **元素附魔类 buff 要可选/可关** —— 避免 Bennett C6 毁物理流的尴尬
3. **新支援角色要找"未占领的 bucket"** —— Furina 找到"反弹治疗增伤"才出圈
4. **限制 stack cap + 互斥关系** —— 鼓励"专精分流"避免"全堆同类"

### 加法增伤区的稀释陷阱（单一加法 bucket 必知）

若项目把所有增伤塞进**同一个加法区** `(1 + Σ增伤)`（属性增伤 + 技能增伤 + 全局增伤 + 弱点增伤… 全相加，原神 DMG% bucket 即此类），角色**自带增伤（自拐）的边际会被面板增伤稀释**：

```
自拐实际边际 ≈ 自拐% ÷ (1 + 已有面板增伤%)
```

- 例：自拐 +24%，面板已堆 +60% 时，实际边际 ≈ 24 ÷ 1.6 ≈ **+15%**（面板越肥，新增伤越不值钱）。
- 想做"不被稀释"的强增伤角色，需要**独立乘区**机制（绕开加法 bucket）；项目若无独立乘区，强增伤角色就有这条天花板，**别按"名义 +24%"估强度**。
- 推论：验证数值必须**用项目真实公式跑**（见 § 十）—— 简化公式会掩盖稀释，让你高估增伤 / 命座边际。

## 五、命座/星魂 5 段设计模板

> 📌 **品类适用性**:本节(命座/星魂 5 段 + 付费哲学)**适用抽卡 / gacha 养成品类**;买断制 / 非 gacha 项目可跳过,或仅借鉴"同一养成轴分梯度做质变跳跃"的通用思路。

米哈游已验证的通用模板:

| 段位 | 设计哲学 | 例子 |
|---|---|---|
| **E0/C0** | 完整体验(必保) | 任何角色 0 命应完整可用 |
| **E1/C1** | 数值小补 + 次要 QoL | 平民优化目标 |
| **E2/C2** | **质变性价比拐点**(故意诱导付费) | Acheron E2 解除累积限制 |
| **E3/E5** | 技能/大招 +2 等级 | 小幅 DPS 提升 |
| **E4** | 副功能数值 | 战术性 buff |
| **E6/C6** | **鲸鱼勋章**(机制翻天覆地) | Bennett C6 火附魔 / 雷电 C6 缩 CD |

**付费哲学**:E0 必保(避免被骂"必抽 E2"),E2 故意质变(诱导付费),E6 是奢侈品。

### 命座 / 养成梯度 · 健康自检阈值（经验基线 · 项目可调）

设计完一条养成轴（命座 / 星魂 / 叠影）的数值梯度后逐项自检，任一越界 → 回调：

| 维度 | 健康基线 | 越界后果 |
|---|---|---|
| 满命总梯度（如 C0→C6） | **+80~120%** | 低 = 养成无感；高 = 逼氪 / 破 PVP 平衡 |
| 逐级梯度 | **累计严格递增**；质变跳点（C2 付费拐点 / C6 鲸鱼）允许更大跳、其余平滑 | 累计倒挂 = 养成节奏怪（入门比中级还肥） |
| 0 命 / 0 叠影 | **完整可玩**，核心循环不依赖养成 | 否则"必抽某命"骂声 |
| 场景梯度（群伤角色的单体表现） | 单体 / 群战 **40~55%** | < 40% = 打单体 Boss 像残废 |
| 爆发（大招 / 强招段） | **5~15 倍**于平均段 | 过高 = 一键秒；过低 = 大招无存在感 |
| 暴击 | CR **不溢出 100%**（留余量）· CR : CD = **1 : 2** | CR 溢出 = 暴击率词条浪费 |
| 加法增伤区膨胀 | **× ≤ ~1.5** | 单一加法 bucket 天然受限，自拐边际被稀释（见 § 四） |
| PVP 满养成 / 0 养成（若有 PVP） | **≤ ~1.8×** | 过高 = 付费碾压 |
| 养成物 / 套装效果（若有） | 单条 4 件套 ≈ **一个质变命座量级**（条件触发 +30~50%），不盖过满命主轴 | 套装 > 命座 = 养成轴互抢 |

> 📌 **按定位 / 品类取用**：「场景梯度」假设群伤主 C、「PVP」「养成物」假设对应系统存在 —— 单体 / 单机 / 无 PVP / 无养成物的角色，对应项直接判 N/A，不强凑。数字是经验基线、按体量调；非 gacha 项目主要取"0 养成完整可玩"+"累计递增"两条。**怎么验证 → 见 § 十**。

## 九、配置基线(项目落地)

### 伤害公式
- **架构**:纯乘法 4 bucket(ATK% / DMG% / CRIT / RES)+ 选择性 snapshot(召唤物 + 持续技)
- **暴击**:双爆 CR:CD = 1:2 词条比
- **抗性**:分段公式,75%+ 软衰减

### 击破系统
- **阈值**:小怪 50 / 精英 100 / Boss 200(基准)
- **窗口**:Boss 10-15 秒
- **倍率**:基础 160%,可堆 300%(参考 FF7R)
- **UI**:头顶第二根条(透明显示,玩家随时看到)

### 命座/星魂(适用抽卡 / gacha 品类;非 gacha 项目可跳过)
- **E0/C0 必保完整**(避免被骂"必抽 E2")
- **E2/C2 质变性价比拐点**(故意诱导付费)
- **E6/C6 鲸鱼勋章**(机制翻天覆地)

### 防国民老婆 4 铁律
- 通用 ATK 不独立 bucket
- 元素附魔可关
- 新支援找未占领 bucket
- 限制 stack cap + 互斥

## 十、数值建模验证（建模 → 对抗验证 → 设计稿内嵌论证）

> 凡是带**数值梯度**的设计（命座 / 养成 / 增伤 / 击破倍率），拍脑袋容易过强 or 过弱。设计稿不该只写"我觉得平衡"，而要给**可复算的论证**。三步：

### 1. 脚本化建模（参数可换）

- 把**你项目的伤害公式**（架构见 [`conventions/TEMPLATE.md`](../conventions/TEMPLATE.md) § 9，公式 5 架构见本文 § 一）写成脚本，面板 / 系数 / 倍率做成**可换参数**，数值策划给真实面板后一键复跑。
- 🔴 **用项目真实公式跑，别用简化公式** —— 简化式会掩盖"加法增伤区稀释"（§ 四）这类只有真公式才暴露的问题，导致高估增伤 / 命座边际。
- 评估**相对强度 / 养成梯度**时，守方相关乘区（防御 / 抗性 / 等级压制）随环境不变，可**归一化为 1**，只比 `技能 × 暴击 × 增伤` 这几个随养成变的乘区。

> 📐 **项目公式填这里**：本方法论不预设你项目的具体公式；把你的伤害 / 击破公式写进 [`conventions/`](../conventions/TEMPLATE.md) 或 [`knowledge-local/project-knowledge/`](../knowledge-local/)，建模脚本就用它。

### 2. 独立 agent 对抗性交叉验证

- 数值结论**别自说自话**。用一个**独立 agent**（不预设你期望结论的）重新建模 / 质疑算法，专门找"哪里高估了 / 哪条假设不成立"，**对抗确认偏误**。
- 命座梯度、增伤边际、击破收益这种"容易自我说服"的地方，交叉验证最值。

### 3. 设计稿内嵌可视化论证

把验证结果**画进设计稿**，让评审一眼看懂强度曲线：

- **离线归档**：设计稿（HTML / md）内嵌 **SVG** 静态图（命座养成曲线 / 逐级边际 / 增伤稀释 / 场景梯度等），脱离环境也能看。
- **即时仪表盘**：对话 / 评审中用可视化工具（如 Chart.js，或任意能出交互图的工具）做交互图，给策划即时看不同参数下的曲线。
- 🔑 **每张图配「三问」小字注解**：**怎么算**（公式 / 参数）/ **为什么**（设计意图）/ **结果**（优缺点 / 是否越界）。没注解的数字图 = 评审看不懂、拍不了板。

### 闭环

建模跑完，用 § 五「命座 / 养成梯度健康自检阈值」逐项过；任一越界 → 回设计调参 → 复跑。**脚本化 + 对抗验证 + 阈值自检**三件套齐，数值论证才算闭环。
