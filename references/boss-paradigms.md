# Boss 设计范式库(V2)

业界 25+ 款 ARPG 调研 + 多个 Boss 实战沉淀。**16 种核心 Boss 范式**——设计新 Boss 时**选其一 / 组合多种 / 或推新型**。

⚠️ **核心方法论**:Boss 设计应该是**"反向设计"**——先问"我要教玩家学什么"再设计 boss 招式,不是反过来(参见 [元规则](meta-rules.md))。

---

## A 型:HP 阶段化(经典范式)

**适用**:常规剧情 boss、章节末 boss、试金石型 boss

### 结构

```
P1 试探期(HP 70-100%) → 基础技能轮转 + 受伤触发循环
P2 高压期(HP 40-70%) → P1 移除部分招 + 新增强招(突进/抓取/连招)
P3 终局期(HP 0-40%) → P2 + 终局机制(护盾/超必杀)
通用狂暴(跨阶段) → 脱力起身后必放大招
```

### 设计要点

- 阶段化是"模组堆叠",不是"换 boss"——保留前阶段招式,**叠加新招而非全换**(类似哈迪斯"加招不换 boss")
- 每阶段 6-7 招(示例帧数预算,30fps:简单招 18-22f / 警示招 25-30f / 大招 35f+)
- 必备 3-5 个 ★ 特殊设计点:**受伤触发循环 + 机制盾 + 视觉预警约定 + 狂暴大招**

### 样本

- 原神若陀龙王(三阶段,每阶段切元素+形态+专属机制)
- 艾尔登半神 boss(普遍二阶段,二阶段加 DOT/AoE/Debuff 同套路扩展)

## B 型:状态机切换(双态)

**适用**:节奏感强 boss、考验玩家弹反技术、剧情角色 boss(原型反派)

### 结构

```
默认态(蓝刀/常态) → 循环段 1: 普攻起手
                  → 循环段 2-A/B/C(33%/33%/33% 概率分支)
                  → 循环段 3: 弹反考核技
              + 应激位移(玩家贴身概率触发,管控距离)
              + 韧性打空 → 虚弱期 → 狂暴态(红刀/觉醒)
狂暴态 → 起手必出招(护盾/全屏 AOE)
       → 红刀攻击循环(高速/连段/末段考核)
       → 时间到回默认态
```

### 设计要点

- 状态切换由**韧性条**驱动(玩家弹反考核技打空韧性 → 虚弱 → 狂暴),不是 HP 阈值
- **概率分支循环**(段 2-A/B/C 三选一 33%)避免节奏机械
- **应激位移**(某招后闪)管控距离,防玩家纯莽攻
- **强表演 + 末段考核**(连砍 5-7 段前段秀+末段红光必弹反)
- 必备"战斗循环时序" sheet(独立 sheet,按段编号)

### 样本

- 卧龙·苍天陨落吕布(马上→下马二阶段 + 化解考核)
- 鬼泣 5 Vergil(Concentration 浓度槽驱动招式强度)
- 战双帕弥什精英 boss(default → enraged 切换)

### 关键 sheet:战斗循环时序

按"段编号 + 概率 + 流转逻辑"组织,比 AI 策略 + 连招策略两表更清晰:

```
段 1: attack_01 普攻 3 连击
段 2-A (33%): skill_01 远程多发
段 2-B (33%): skill_02 召唤
段 2-C (33%): skill_03 大范围扫
段 3: skill_04 弹反考核技
回循环: 玩家弹反 skill_04 削韧打空 → 进入虚弱+狂暴
```

## C 型:机制破解型 🆕

**适用**:团本/Raid boss、有"小队配合"要素的 boss

### 结构

```
boss 默认免疫常规伤害 / 高减伤
→ 玩家必须完成"机制"才能进入 DPS 窗口:
   - 打 4 根柱
   - 收集 N 个核心
   - 让 boss 进入特定状态
   - 命中 boss 的弱点部位
→ 进入 Pressure / Stagger 窗口 → 高倍率输出
→ 窗口结束回到免疫态,机制重置
```

### 设计要点

- **"机制 → 窗口"二段式**,玩家要先解机制再输出
- 机制可设计为**特定职业专属任务**(让某个 enabler 职业有用武之地)
- 窗口期 8-15 秒,倍率 1.5-3 倍
- 适合"配队策略",不是单玩家硬刚

### 样本

- Relink Proto Bahamut(召唤元素塔 → 破塔 → 眩晕 → 进攻窗口)
- FF7 Rebirth Sephiroth Reborn(必须先打碎脸或核心来触发 Pressure)
- FF7 Rebirth Gilgamesh(用 perfect block 让他掉武器/掉盾 → 破防窗口)
- 鬼泣 5 Vergil(Concentration 浓度槽,玩家原地不动+精准操作 → 攻击范围/伤害变化)

### C+ 变体:博弈式机制反制(每件装备 = 一个"做/不做都有后果"的博弈点)🆕

C 型的进阶落地。把 Boss 的**每件招牌装备/招式**做成一个**反制博弈点**:**不反制有代价 + 反制有奖励**,逼玩家主动学,而不只是"不解机制就没 DPS 窗口"(纯 C 型只有"不解 = 0 收益"的单向门槛)。

**结构**:
```
Boss 一件装备 / 一个招(护盾 / 大招 / 蓄力):
  ├─ 玩家不反制 → 明确代价(Boss 回血 / 强化 / 高伤 AOE / 拖战斗)
  └─ 玩家反制 → 明确奖励(Stagger / 自损 / 永久解除该机制 / 评级)
       └─ 反制还可分层:基础反制(小奖励) / 完整反制(大奖励 + 永久解除)
```

**设计要点**:
- **每个博弈点都要"两边都有明确后果"** —— 不反制不是"白嫖安全",而是有代价;反制不是"必须做",而是"做了更赚"
- 反制收益可做**完整曲线**(击破 0/1/2/3 节点对应不同回血量 + 奖励),不是"全或无"硬墙
- **"永久解除"是高价值奖励** —— 完整反制后 Boss 本场失去该机制,玩家获得"我把它拆了"的掌控爽感
- **反制方式跨角色通用**,用动词描述不锁特定角色
- 多个博弈点 = **多条独立反制线** = 多解法 Boss(暴力流 / 反制流 / 混合流)

**样本**:
- 苍蓝档案 Total Assault(Groggy Gauge 破韧 ↔ Active Gauge 倒计时双计量博弈)
- FF7 Rebirth Gilgamesh(perfect block 让他掉武器 = 反制奖励)

## D 型:距离行为树 / AI 生态 🆕

**适用**:开放世界 boss、PVE 大型怪、有"猎人感"的 boss

### 结构

```
默认态 → AI 持续观察玩家行为:
  - 玩家贴脸(<3 格)→ 切近战 AI(轻击+抓投+后闪)
  - 玩家拉远(>4 格)→ 切远程 AI(投射+召唤+突进)
  - 玩家不躲多次 → 进入 enraged(攻速+20%,招式更狠)
  - 玩家打多次同部位 → 部位"流血→折断→不能再用该招"
  - 30 秒无攻击 → exhaust(疲劳:攻速降低,流口水)
  - 玩家逃跑距离过远 → 瘸腿逃跑到巢穴睡觉,可捕获
```

### 设计要点

- Boss 像**生命体**不像状态机
- AI 不为玩家服务,**有自己生活**(疲劳/愤怒/逃跑)
- "部位破坏"机制:**打多次同部位** → 折断/出血 → boss 失去该招
- 可有"Turf War"分支(boss 遇到别的 boss 互殴,玩家可利用)

### 样本

- MH 系列(愤怒/疲劳/瘸腿/Turf War,生态化经典)
- 红色沙漠 Reed Devil(贴脸切近战 / 拉远切远程)
- 对马 AI(玩家秒杀队长后小怪 50% 几率"恐惧逃跑")
- 黑神话 Boss(亢金龙塔身爬战 + 动态行为)

## 多解法 Boss 设计原则 🆕

借鉴 **DD2 Talos** 范式(8 个水晶突起可被多种方式破):

任何 boss 都应该至少支持 **2-3 种通关路径**——不要"唯一正确解"。

**多解法示例**(以"高大型 boss"为例):
- 远程武器风筝(慢但稳)
- 攀爬到弱点近战(快但险)
- 用召唤物拖住,本体输出(配队策略)
- 利用环境陷阱(场景互动)
- 用控制链满后团队联动技 burst(团队配合)

**好处**:
- 玩家"打通"成就感不同(我用 X 流派打过的)
- 多周目重玩动力(下次试别的解法)
- 反"必须特定角色"陷阱(参见反模式 8)

**设计要点**:
- 至少 2-3 种通关路径
- 每种路径有不同的"看点"和"挑战点"
- 没有"次优 30% 效率"的废路径——所有路径都 60%+ 效率
- 设计师测试时用"非主流配队"试通关——通了 = 合格

## Boss 设计的"必填"要素

无论选哪种范式,以下都是必备:

| 要素 | 说明 |
|---|---|
| **3-5 个 ★ 特殊设计点** | 必有 1 个"美学+独家招式+玩家会取外号"的招 |
| **教学目标** | 这个 boss 强迫玩家学什么机制? |
| **每招 telegraph 清晰** | 简单招 18-22f 前摇 / 警示招 25-30f / 大招 35f+(示例帧数预算,30fps;哈迪斯哲学) |
| **视觉预警约定** | 至少 1 个必闪/必弹反招,配清晰的预警视觉(红光=必闪 只是一种示例约定,项目应定义自己的 telegraph convention) |
| **彩光语义化** | 不同颜色对应不同应对(示例约定,语义由项目定义) |
| **架势条 / Stance Break** | 头顶第二根条,削破 → 暴击窗口(示例约定;是否设架势 / 击破系统由项目定义) |
| **击破职业的工作位** | 若项目设有"击破/破韧"定位,Boss 设计要给它留"用武之地" |
| **应对方式可读** | 每招都要有"玩家解法",不要"魂系随机大招"(只狼哲学) |
| **复用 vs 新增标注** | 约 80% 招复用角色资源,约 20% 新增是 ★ 招(美术成本管理) |

## 同 IP 角色 ↔ Boss 资源复用方法论

当同一 IP / 同一世界观既出**玩家角色**又出 **Boss**,两者可共享一套美术资源以控成本:

- **同 IP 的玩家角色与 Boss 可共享动作 / 受击资源**(普攻、特殊技动作,以及 idle / 受击 / 倒地 / 起身等通用受击集),Boss 只为机制专属招(护盾 / 空袭 / 瞬移等无对应招式)新增资源。
- **发射物(子弹 / 导弹 / 激光)是独立资源,可一鱼多吃** —— 子弹本体(弹体 + 飞行 + 落地爆炸)做一套,被多招复用;"发射动作"各招另算。资源表把子弹单列一行。

## 设计新 Boss 的起手流程

1. **确定教学目标**:本 Boss 要让玩家学什么机制?
2. **选基础范式**(16 种中选 1-2 种组合)
3. **从灵感种子推导 ★ 招**(参见 [元规则](meta-rules.md) 机制创新法则)
4. **画战斗循环时序 / 阶段表**——B 型用循环时序,A 型用阶段表
5. **列★点**(3-5 个,至少 1 个"被玩家取外号"招)
6. **填技能详情表 + AI 策略 + 连招策略**
7. **拆动作需求 + 特效需求 + 程序需求**
8. **对照 [元规则](meta-rules.md) 做强度评估**——对照你的强度评估清单,命中条目偏少就改设计

## 候选未沉淀范式(简短列表)

调研中看到但未充分提炼:**资源消耗型**(玩家每次输出都消耗某资源) / **召唤潮型**(boss 持续召小怪) / **真随机型**(招式真随机,逼"看招辨型"非"背板") / **环境吞噬型**(场地越来越小)。
