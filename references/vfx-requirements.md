# 特效需求规范

ACT 战斗特效设计的通用方法论:如何把一招的视觉表现拆解、分级、命名、并对美术下需求。
本文给出一套可直接套用的骨架;其中的**命名前缀、帧率、颜色约定都是示例**,各项目按自己的资产规范替换即可。

> 全文用一个 IP 无关的示例角色 **苍岚(CangLan)** 贯穿举例:薙刀女武者,近战爆发,核心机制「势」蓄力层(长按特殊技积「势」0–4 层,松手按层数释放对应威力斩击,满 4 层解锁「破势·一闪」)。苍岚所属示例项目用 **60fps**;视觉预警契约 = **金色闪光 = 可格挡 / 紫色闪光 = 必闪**;特效命名规范 = `Vfx_<角色>_<招式>_<段位>`。这些都是该示例项目的选择,**不是通用默认**——你的项目用什么招式集 / 命名 / 帧率 / 预警色,在 [conventions](../conventions/TEMPLATE.md) 里自己定。

---

## 一、三段式拆分(核心方法)

每招特效拆三段:

| 段 | 时机 | 视觉关键词 | 备注 |
|---|---|---|---|
| **前摇蓄力** | 招式前摇阶段 | 凝光、绕身光环、地面预警圈、角色身周能量汇聚 | 给玩家"看到→反应"窗,警示作用 |
| **命中瞬间** | 攻击判定生效那 1–3f | 拳风、刀光、冲击波、裂纹、镜头微震、屏幕泛白 | 锐利、单段、强冲击,**强度峰值** |
| **后摇余韵** | 招式后摇阶段 | 余烬、尘土散开、能量逸散、喘息 | 缓慢消散,玩家反击窗 |

实例(苍岚满势特殊技「破势·一闪」`Skill_Charge_Release_Lv4`,帧值按 60fps 示例,按你项目帧率替换):
- 前摇 0–14f:**凝势起手**(薙刀垂地,身周「势」能量螺旋汇聚 + 脚下聚光 + 地面预警弧线,给玩家看预警走位)
- 命中 4f:**一闪斩**(青白长刀光横扫贯穿 + 大范围裂纹 + 强镜头震 + 屏幕泛白 + 短暂定格)
- 后摇 0–18f:**收势余韵**(刀光残辉逸散 + 身周尘土 + 收刀姿态)

**例外**:位移 / 后撤等非攻击招只需"起手瞬间"一段(如闪避 = 脚底擦地烟尘 + 残影,1 段)。

## 二、命名规范(示例命名,定义你自己的)

命名的唯一目标:**一看名字就知道是"谁的、哪一招、哪一段"**。前缀 / 分隔风格各项目自定;下面用苍岚示例项目的方案 `Vfx_<角色>_<招式>_<段位>` 举例。

### 玩家角色特效(示例格式)

`Vfx_<角色>_<招式>_<段位 / 可选>`

实例(苍岚,招式集见她的 [设计稿](../output/characters/cang-lan/md/苍岚-战斗设计.md);**她的项目只有这些招,不是每个项目都一样**):
- `Vfx_CangLan_Attack_01` — 普攻 1 段横斩刀光
- `Vfx_CangLan_Attack_01_CanYing` — 普攻 1 段挥刀残影
- `Vfx_CangLan_Charge_Loop` — 凝势蓄力(loop,随「势」层 1→4 增亮)
- `Vfx_CangLan_Skill_Release_Lv1` … `_Lv3` — 斩势释放(威力随层递增)
- `Vfx_CangLan_Skill_Release_Lv4` — 满势「破势·一闪」专属
- `Vfx_CangLan_Dodge_Perfect` — 完美闪避残影 + 时停涟漪
- `Vfx_CangLan_Parry_Success` — 弹反成功金属火花
- `Vfx_CangLan_Ultimate` — 大招「奔雷」多段雷斩 + 终刀全屏冲击
- `Vfx_CangLan_Execute` — 处决斩落

> 注意:苍岚没有"团队联动技 / 协同技"特效,因为她的项目里联动不是每角色必备招(切人联动是项目级机制)。**别假设每个角色都有某类招**——按你项目的招式集来。

### Boss 特效(示例格式)

Boss 招式常按编号(`skill_01 … skill_N`),特效同形:`Vfx_<Boss缩>_<招式编号>_<段位>`。

实例(某双形态机甲 Boss,缩写 `MJ`):
- `Vfx_MJ_Skill01_Combo` — 近战普攻三连
- `Vfx_MJ_Skill04_Counter` — 必闪 / 可弹反招的对撞
- `Vfx_MJ_Skill06_Buff` — 自增伤(3 段叠加)
- `Vfx_MJ_Skill10_Charge / Knockback / Shield / Break` — 护体形态 4 段子特效(蓄力 / 击退 / 护盾 / 破盾)
- `Vfx_MJ_Skill11_Takeoff / Pre / Main / End` — 大招 4 段子特效(起飞 / 蓄势 / 俯冲 / 收尾)

### 通用受击特效(示例格式)

`Vfx_<角色>_Attack_Hit_0N` 或 `Vfx_<角色>_Attack_Hit`(通用):
- 苍岚:`Vfx_CangLan_Attack_Hit`(刀光通用受击)
- 钝器型角色:`Vfx_<角色>_Attack_Hit_01`(拳头受击) / `_02`(踢腿受击)

## 三、优先级(P0 / P1 / P2)

| 优先级 | 含义 | 何时用 |
|---|---|---|
| **P0** | 必做,角色 / Boss 基础体验靠它 | 普攻全部 / 核心机制招全部 / 大招全部 / 受击通用 |
| **P1** | 重要但可后置 | 弹反 / 格挡特效 / 招式副段 /(若有)联动技核心 |
| **P2** | 锦上添花,可后期补 | 起跳 / 落地 / 大招地面附加 / 收尾余韵 |

设计时**默认所有招都从 P0 起手**,美术资源不够再降级到 P1/P2。

## 四、字段表(美需特效 xlsx)

每条特效一行:

| 字段 | 说明 |
|---|---|
| 序号 | 顺序号 |
| 优先级 | P0 / P1 / P2 |
| 特效名称 | 中文描述,如"【普攻】横斩刀光"、"【特殊技】地面积势过程" |
| 对应动作 | 资产名(`Attack_01` / `Charge_Loop` 等),关联到动作需求 sheet |
| 特效命名 | 你的资产命名(示例:`Vfx_<角色>_<招式>_<段位>`) |
| 特效描述 | 1–2 句话讲清楚视觉效果 |
| 参考图 / 参考视频 | 图文 / 视频链接 |
| 时长 | 跟随动作 / loop 循环 / N 帧(含生成及消散) |
| 范围 | 横X × 纵Y / 包围身体 / 跟随子弹 / 屏幕级 |
| 特效类型 | 技能特效 / 子弹 / BUFF / 表演 |
| 备注 | 关联说明、特效师注意点 |
| 制作进度 | 动作就位时间 / 特效制作时间(待 / 日期) |

**特效类型详解**(四分类):
- **技能特效**:跟随角色动作,如拳风、刀光、爆炸、地面冲击
- **子弹**:用作子弹的特效,**原点位置不带偏移**,程序用子弹逻辑生效(如飞出的剑气、飞出刀光)
- **BUFF**:状态 / 标记类,常 loop,跟随目标(如命中标记 loop、Boss 护盾层数 UI)
- **表演**:演出 / 过场,如大招前摇起飞、破盾爆裂

## 五、Boss 特效 sheet 顶部风格说明

Boss 特效 sheet 顶部固定加风格说明段:

```
特效需求(按"<风格关键词>"风格)
重点:
① 前摇蓄力 / 命中瞬间 / 后摇余韵 三段式拆分
② 角色人设——<风格描述,如"重装人设——拳风偏厚重、有重量感">
③ 预警招用项目统一的视觉预警提示,与玩家格挡 / 闪避约定一致
④ 复用项尽量沿用现有特效资产,仅做颜色 / 强度调整
```

示例(苍岚所属项目,视觉契约 = 金色闪光=可格挡 / 紫色闪光=必闪):
> 特效需求(按"凌厉一闪 / 长柄横扫"风格)
> 重点:① 三段式拆分;② 武者人设——刀光偏锐利、有"破空感";③ 预警招用统一视觉提示(金=可格挡 / 紫=必闪),与玩家格挡 / 闪避约定一致;④ 复用项尽量沿用现有特效资产,仅做颜色 / 强度调整

> 关于"预警招的颜色契约":各项目应定义自己的**视觉预警契约 (telegraph convention)**——某种颜色的闪光 / 锁定提示对应"必闪"或"可弹反 / 可格挡"。示例中「金=可格挡 / 紫=必闪」只是一种方案;"红光=必闪"也是常见方案之一。关键是**全游戏统一**,且特效与战斗判定一致(见 [设计审美](design-aesthetics.md) 招式可读性维度)。

## 六、设计特效需求时的起手

1. **先看动作需求 sheet**,把每个资产名(`Attack_01` / `Charge_Loop` 等,均为示例命名)都列出来
2. **每个资产配 1–3 条特效**:基础招 1 条,复杂招 2–3 条(如蓄力体的 pre + charge + release 各一条;大招前摇 + 命中 + 后摇各一条)
3. **复用 vs 新增**标注:能用现有资产改色就标"复用资源制作",否则"新增"
4. **优先级**:核心招(普攻 / 核心机制招 / 大招)默认 P0;弹反 / 格挡副段 P1;起跳 / 落地等 P2
5. **跟动作 sheet 双向校验**:动作 sheet 有的资产,特效 sheet 应都覆盖;特效 sheet 引用的"对应动作"必须在动作 sheet 找得到

---

## 七、特效需求质量门禁

提交前自检:

- [ ] 每个动作资产至少 1 条特效
- [ ] 命名遵守项目统一的资产命名规范(示例:`Vfx_<角色>_<招式>_<段位>`)
- [ ] 优先级标注完整(P0 / P1 / P2)
- [ ] 复用方式标注完整(复用 / 新增)
- [ ] 三段式拆分到位(前摇蓄力 + 命中瞬间 + 后摇余韵 至少各 1 条)
- [ ] 预警招在前摇段明确写视觉预警(如"<契约色>锁定" / "<契约色>蓄势")
- [ ] 风格说明段在 sheet 顶部(Boss 必有,玩家角色可选)
