# 效果与养成设计文档范式

给一个**已有基础设计**（动作 / 招式 / 核心机制）的角色，单独展开「**效果 + 养成 + 数值**」这一层时，用这套结构产出一份**审阅级 HTML**。

> **适用**：gacha / 抽卡品类（含命座 / 养成物 / 属性词条）。买断 / 非 gacha 项目取适用子集（技能附带效果 + 属性投放 + 数值验证），跳过命座 / 养成物章。
>
> **跟角色主设计稿的分工**：基础动作 / 招式 / 资源清单在角色主设计稿；本范式只管「效果挂什么、养成解锁什么、词条怎么堆、数值健不健康」。

---

## 一、文档结构（7 章）

| 章 | 内容 | 适用 |
|---|---|---|
| 壹 设计说明 | 角色定位 / 本稿范围 / 跨语境说明 | 通用 |
| 贰 技能附带效果 | 每招挂的 buff / debuff / 标记 / 状态（招式 → 效果表） | 通用 |
| 叁 命座 / 养成解锁效果 | 按 5 段付费哲学（C0 完整 / C2 拐点 / C6 质变），每段解锁 / 强化一个效果 | gacha |
| 肆 养成物套装效果 | 2 / 4 件套，4 件套 = 流派核心 | gacha |
| 伍 属性投放 · 资源 · 操作循环 | 词条优先级（走哪条赛道、为何不堆某词条）+ 资源经济 + 短轴 / 长轴循环 | 通用 |
| 陆 数值建模验证 + 边际效应 | 养成梯度建模（图）+ 边际分析 + 健康自检 | 通用 |
| 柒 最终总结 | 效果 / 养成 / 数值三层闭环 + 待办（如 PVP 复核） | 通用 |

## 二、3 类图示（数值 / 机制可视化）

每个有梯度 / 张力 / 流程的地方配一张图，胜过纯文字表：

1. **养成曲线** —— 命座 C0→C6 累计强度指数 **SVG 折线** + 健康区带（§五 +80~120%）+ 质变跳点高亮（C2 / C6）。
2. **操作流程图** —— 短轴 / 长轴 **box + arrow** 流程，核心爆发点（如处决招）用异色框标出。
3. **边际张力图** —— **双轴**（收益↑ vs 代价↓），如「势层：伤害倍率↑ / 剩余移速↓」，一图说清「越强越脆」类核心张力。

> 图用内嵌 SVG（离线归档），冷蓝配色贴 `blank-design-doc.html`；对话内可先用可视化工具（Chart.js）做即时仪表盘给策划看，再固化成 SVG。

## 三、数值注解四要素（每个数值点配小字）

凡是出现「算出来的数」（命座梯度 / 倍率 / 边际 / 词条比），旁边小字配齐**四要素**，缺一不可：

| 要素 | 写什么 |
|---|---|
| **怎么算** | 公式 / 方法（如：每命座效果折算 DPS%，在 C0 = 100 上累乘） |
| **为什么这么算** | 设计意图（如：C2 / C6 做付费 / 鲸鱼跳点） |
| **算的过程** ⭐ | **实际演算步骤**，别只给结论（如：C1 +5% → 105；C2 +12% → 118；… C6 +37% → 195） |
| **结果** | 结论 + 是否越界（如：195 ÷ 100 = +95%，落健康区） |

> 「算的过程」最容易漏 —— 评审要能**复算**，不能只看到结论数字。标注基准（如「倍率以 1 势斩势 = 1.0 为准」），避免同一指标多基准打架。

## 四、落地（复用交互层，别另起炉灶）

1. **复制 [`templates/blank-design-doc.html`](../templates/blank-design-doc.html)**（完整审阅交互层：评审备注 / 导出回传 / 改动历史 / 两级目录）。
2. **7 章重标** + 填本范式内容；**删用不到的章 / Tab**（同步 `SECTIONS` 数组，见 [review-html-doc.md](review-html-doc.md)）。
3. **改 `LS_PREFIX` / `<title>` / 封面**；**协作说明改成评审向**（本范式无资源清单，去掉资源协作列那套）。
4. **数值用相对指数 / 倍率（归一化）**，绝对伤害代入项目真实面板；验证对照 [numbers-and-progression.md](numbers-and-progression.md) **§五**（健康自检阈值）+ **§十**（脚本建模 → 对抗验证）。

> ⚠️ **不要复制成独立 blank 模板** —— 交互层（约 4500 行）会跟 `blank-design-doc.html` 漂移。本范式 = **文字模板**，交互基座唯一源自 `blank-design-doc.html`。
