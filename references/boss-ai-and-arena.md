# Boss AI 与场地参与战斗(工程化基线)

> 本文是「Boss AI 与场地参战」的工程化方法论,从业界多款商业 ACT/ARPG 调研沉淀而来。内容为纯通用工程知识,可直接落地到你的项目,作为「Boss AI 状态机 / 场地参与战斗」的设计与配置参考。

⚠️ **本文件性质**:与 [打击感技术](hit-feel-tech.md)、[数值与养成](numbers-and-progression.md) 同性质——**工程化技术规范**,落地到你的项目时可作为「Boss AI 状态机 / 场地参与战斗」的配置参考。

业界 8+ 款 ACT/ARPG 调研沉淀。填补 [Boss 范式库](boss-paradigms.md) 「AI 工程化怎么写 + 场地怎么参与战斗」的细节缺口。

## 一、🌟 Boss AI 设计 10 条工程化原则

### 1. 主体驱动 vs 玩家响应双轨

Boss **不是"等玩家来打的木桩"**,要有自己的状态主轴(捕食/游荡/激怒/疲劳/跛行)。**MH Wilds 生态化** 是标杆。

### 2. 状态切换 = HP × 累积事件 × 时间 × 玩家行为 四维

单纯 HP 阈值切换显得机械(玩家会算血量"挂机");叠加累积 stagger 次数 + Active Gauge 时间 + 动画 ID 检测 → 节奏感涌现。

### 3. 招式权重 = 距离 × 角度 × 玩家状态 × 阶段 × cooldown 五维表

每招有触发条件,多候选满足时再随机。这五维权重表是工程实现的核心数据结构。

### 4. 🌟 动画感知 ≠ 输入感知(FromSoft 工程关键分水岭)

检测玩家"**已开始的动作 ID**"(挥剑/喝瓶/施法)而非"按键事件"——既保留 AI 反应速度,又规避"读输入作弊感"。玛莲妮亚抢喝瓶用突进刺杀正是此机制。

### 5. 危险招分级 + 视觉提示分级

必闪 / 可弹反 / 可见切 / 可 Mikiri 不能混色。**前摇 14-28 帧标准化反应窗口**:
- 只狼"危"红字 ~14f(见切)
- 法环红光 ~28f(必闪)

### 6. 应激位移即性格

玩家贴脸时 boss 是后撤 / AOE / 抓技 / 不动?玩家拉远时是追击 / 远程 / 等待?**响应表是 boss 性格的核心载体**:
- 力量型 → 偏抓技
- 技巧型 → 偏后撤
- 远程型 → 偏不动 / 远程压制

### 7. 资源槽双向耦合(Vergil 浓度证明)

资源槽不必只是"玩家增强工具"——boss 自己用同套规则,玩家通过自己玩法间接调节 boss 表现,**形成"行为镜像"**。

### 8. 招数数量取决于学习曲线位置

- **早期 boss**:3-4 招(广智)→ 教夯实闪避时机
- **中期 boss**:10-20 招 → 综合技能考验
- **终局 boss**:50+ 招(二郎神 73 / 苇名一心 30+ / Gna 综合考试)→ 集大成考核

**别给所有 boss 都堆招式** —— 招数多不等于难度高。

### 9. 场地几何 = boss 招式空间约束

Boss 设计师必须先确认"我的最大 AOE / 突进距离 / 召唤物范围",关卡设计师按此设计场地。**圆形大场 + 留 1-2 反常规通道** 是 FromSoft 半神 boss 范式。

### 10. 学习型 AI 不必显式

招式池随 HP / 阶段开锁 + 动画 ID 感知的组合,**已能制造"AI 在学习我"的错觉**。真要做学习型 AI,统计窗口动态调权(玩家持续弹反 → 提升危技比例)是低成本方案。

## 二、8 款 Boss AI 详细拆解

### 1. MH Wilds(生态化标杆)

**5 状态主体行为**:Roaming / Enraged / Exhausted / Limping / Turf War
- **激怒**:累积伤害触发,**不同种类用不同信号**(咆哮 vs 蹦跳,按 monster 个性区分)
- **疲劳**:激怒满后强制,主动找食物 → 玩家喘息窗口
- **跛行**:将死时跛足回巢,**地图叙事性结束**
- **Turf War**:Wilds 去脚本化,基于两怪状态+环境+接触位置**自发触发**
- **应激位移**:玩家贴脸"抖落+后跳",拉远主动靠近

**工程要点**:状态切换 = "主体驱动 + 外部 trigger" 双轨,**不单纯响应玩家**。

### 2. 艾尔登玛莲妮亚 / 戈弗雷(动画 ID 阅读)

- **水鸟乱舞触发**:50% HP + 50% 瞬时伤害 + "**input baiting**"(检测玩家追击动画 → 触发)
- **戈弗雷 → Hoarah Loux 切换**:HP 50% + 必须等戈弗雷上一动作收尾(避免插入式状态机突变)
- **招式权重**:距离 × 角度 × 玩家状态三维表(短距 grab/横扫/跳劈,中距突进,远距魔法/跳跃接近)
- **学习型 AI**:**动画 ID 阅读**(检测喝瓶/施法/举大剑触发对应反制)

### 3. 只狼(危技 AI 选择)

**三类危技**:
- **刺(Thrust)**:可侧步/弹反/Mikiri Counter(forward dodge 踩剑柄,对 posture 大伤)
- **横扫(Sweep)**:必须跳起+落地踢头
- **抓(Grab)**:必须翻滚,弹反无效

**选择逻辑**:概率权重 + cooldown 闸门。Genichiro / Owl Father 把危技嵌入连段第 3-4 段,**强迫玩家不能机械弹反**。

**Owl Father 双阶段**:Phase 1 主动后撤 + 老兵招;Phase 2 召唤幽魂枭,招式池整体替换。

### 4. 黑神话(AI 节奏带与转身)

- **广智(早期)**:3-4 招循环,**教夯实闪避时机**
- **二郎神(终局)**:**73 招** lore-accurate 设计,70% HP 韧性回满 + 招式池扩展,50% HP 加雷攻
- **亢金龙**:玩家爬塔身打 boss,**boss 身体即战场**(见场地模式 2)
- **招式权重**:低 HP → 远程/AOE 飙升;玩家连击中 → 高优先级反击招;连接 finisher → 必现重击

### 5. 战神 Gna / Heimdall(预知型 AI)

- **Gna**:吸收前 9 女武神所有招式 + 增强版,**相同前摇映射多种 follow-up**(玩家无法"前摇匹配反制"简单策略)
- **Heimdall**:**对玩家所有正常攻击 100% 闪避状态机**,只有 Draupnir 长矛能 bypass
- **工程化解读**:Heimdall 验证"**boss 的'难'可以是'必须用对方法'而非'必须打到对位高'**"——puzzle gate 而非数值墙

### 6. 苍蓝档案 Total Assault(双 Gauge 标杆)

**双计量条耦合**:
- **Groggy Gauge**(玩家削):破韧条,失败累积值跨次保留
- **Active Gauge**(boss 自走):3 个黄方格倒计时,满 → boss 大招
- **必须破韧才能进 Active 攻击窗口**:很多 boss 韧条破前有 hyper armor → **必须把破韧时机卡在 Active Gauge 接近满前**

**工程要点**:**用双计量条把 boss 行为从"招式池随机"转成"可规划的 timing puzzle"** —— 对回合制/半自动 ACT 极有借鉴价值。

### 7. 鬼泣 5 Vergil(行为镜像)

- **3 级浓度**:站立/慢走/Trick → 涨条;闪避(精准 ifr)→ 大量涨条;跑动/受击/落空 → 掉条
- **浓度影响**:攻击范围 / 伤害 / 解锁招式
- **boss AI 镜像**:玩家持续位移 → Vergil 浓度低,招式短;玩家敢站定+精准闪避 → Vergil 浓度高,招式范围大但开窗也大
- **工程要点**:**资源槽不只用于增强,可以用于让 AI"读懂"玩家风格再差异化响应**

### 8. 崩铁 Aventurine(回合制 AI)

- **赌石 / All or Nothing**:每两回合开 Gamble,P1 召 2 骰 / P2 召 4 骰
- **赢**:角色秒充满大招 + 当回合 DMG amp
- **输**:几乎必现"囚禁"
- **AoE vs 单体队伍差异巨大** → 队伍编成 → 命中机制差异
- **工程要点**:**回合制 boss 靠"机制门槛 + 信息博弈"而非"招式数量"**

## 三、🌟 场地参与战斗 8 种模式

### 1. **变形场地**(Hades 2 Chronos 时钟)

场地物理结构动态变化(时钟指针顺时针匀速旋转,碰到受伤;倒计时安全区是某数字附近的光圈)。**场地变形 = boss 招式的延伸**。

**几何对齐**:安全区位置 = 时钟数字 = 玩家走位空间 —— 关卡设计师做几何,boss 设计师做时机。

### 2. **Boss 即场地**(黑神话亢金龙塔身爬战)

玩家在 boss 身体上战斗(Shadow of the Colossus 全套范式)。**Boss 几何 = level 几何**,boss 设计师与关卡设计师身份重合。

### 3. **战斗类型切换**(FF16 Bahamut ACT → 射击)

场地变化触发战斗系统切换。**需要两套独立战斗系统并行开发**,跨系统协调视觉语义一致性。

### 4. **环境武器化**(DOA 6 三层 hazard / 法环坠落)

玩家利用场地伤害 boss / boss 利用场地伤害玩家:
- **DOA 6**:墙壁破坏过渡、爆炸箱、电网绳、市民助攻、突坠/缓坠
- **法环**:悬崖坠落、骨灰召唤
- Hazard 触发条件 + 伤害值由关卡设计师定,专属起爆连由 boss/角色设计师定

### 5. **平台/高低差**(Dead Cells Conjunctivius 四平台 / 永劫钩锁)

场地三维化:
- **Conjunctivius**:1 底中央 + 2 侧高位 + 1 顶中央,**末段 boss 摧毁所有平台**(场地崩溃 = 阶段标志)
- **永劫无间钩锁**:所有面墙/屋脊/树枝都可钩 → 完全消解距离优势

**几何精算**:平台间空隙恰好"危险但可躲",弹幕落点 + 武器射程联调。

### 6. **次级 AI 增援**(法环骨灰 / Strangers of Paradise)

玩家可在场地中召唤/引入第三方 AI —— 把"场地"扩展到"动态 entity"。

### 7. **场地破坏过渡**(GoW Stranger/Baldur 战)

战斗中破坏屋顶/墙体进入新场地。**boss 动画段触发 cutscene + 多个相连场地准备**,是高密度合作。

**两种设计哲学对比**:
- **小场强叙事**:密闭 + 破坏物 + 摄影机拉近 = 电影感(初见 boss / 剧情高潮)
- **大场强机制**:开阔无障碍,聚焦招式 reading(综合考试 boss)

### 8. 🌟 **场地节奏门控**(赌石倒计时 / Chronos 倒计时 / BA Active Gauge)

场地不是几何变化,而是**时间维度上的可计算门槛**。把回合制/半自动 ACT 的"节奏感"做出来。

## 四、玩家 AI 队友设计 4 类

### 1. 性格驱动型(DD2 Pawn Inclination)

**4 类 Inclination**:Kindhearted / Calm / Simple / Straightforward
- 性格解锁专属动作(治疗倾向 / 拾物倾向 / 优先打弱怪)
- **跨玩家学习**:Pawn 跨玩家流通 → 累积"知识"(任务点/怪物弱点) → 回来后能引路、提示
- 工具调整:**Rift Incense 改变 inclination**(不是数值堆叠,是性格调换)

### 2. 可控弱化型(MH Wilds Support Hunter)

- **AI 比真人慢**(有意,避免"AI 抢功""玩家无作用")
- **可禁特定行为**:玩家可禁 AI 破伤 / 骑乘 → **保留成就感动作给玩家**
- 难度自适应,proactively 支援(陷阱/回血/药递送)
- **SOS Flare 多重选项**:仅 AI / 仅真人 / 混合

### 3. 配角辅助型(二游 NPC 队友)

**核心原则**:**玩家是英雄,NPC 是配角**——视觉、伤害、决策权重都向玩家倾斜
- 不让玩家替队友失败(NPC 无敌或独立 HP)
- 辅助型行为(buff/嘲讽/定点攻击,不抢主输出)
- 技能特效/伤害数字优先级低于玩家角色

### 4. 半自动型(Blue Archive Striker)

- 全队自动战斗 + 自动出普通技能
- **玩家只控 EX 大招的时机和位置**(拖拽放置)
- 共享 cost points 资源池
- **优点**:单手单指可操作 → 极限契合移动端
- **缺点**:AI 自动战斗不可关,普通技能 AI 决策可能不是最优

### 5. 单人 vs 团队 AI 队友差异

| 维度 | 单人模式 | 团队模式(替人位) |
|---|---|---|
| 效率 | 故意低于真人(留主角光环) | 需接近真人下限 |
| 行为复杂度 | 可做剧情专属动作 | 必须用通用 build/招式 |
| 决策权 | 玩家强干预(指令/偏好) | 决策完全 AI |
| 死亡处理 | 通常无敌/不影响胜负 | 同真人可挂掉 |
| 个性 | 强(Pawn Inclination) | 弱(模拟普通队友) |

## 五、关卡设计师 vs Boss 设计师协作模式

### 5 大协作原则

1. **Boss 招式空间先验** —— boss 设计师先报"最大 AOE 半径 / 突进距离 / 召唤物布置",关卡设计师据此画几何。**不要先做场地再塞 boss**。
2. **共享数据资产** —— 场地 hazard 触发条件、可破坏物 ID、平台坐标必须是共享 xlsx/json,不放在某一方私表。
3. **分阶段同步** —— Phase 1 招式 ↔ Phase 1 场地形态;Phase 2 招式 ↔ Phase 2 场地形态;每个阶段切换是双方共同验证 milestone。
4. **几何对齐** —— 时钟数字位置 = 安全区位置 = 玩家走位空间;**玩家可达的所有 spatial input 都被场地涵盖**。
5. **专职 Encounter Designer**(Naughty Dog / Sony Santa Monica 范式)—— 介于 boss 设计师和关卡设计师之间的新型角色,统合"场地几何 + boss 招式 + 摄影机 + 触发器"。

## 六、落地到你的项目

### Boss AI 状态机模板

```
Boss AI 状态:
├── HP 阶段:P1/P2/P3/狂暴(基础阈值切换)
├── 累积事件:stagger 次数 / 应激位移次数 / 受击连击数
├── 时间维度:Active Gauge(boss 自走必杀倒计时)
├── 玩家行为感知:动画 ID 阅读(不读输入)
└── 招式权重表:距离 × 角度 × 玩家状态 × 阶段 × cooldown
```

### 场地参与战斗清单(Boss 战必填)

- [ ] 场地几何是否能容纳 boss 最大 AOE?
- [ ] 是否有 1-2 个"反常规通道"(给 high-skill 玩家突破)?
- [ ] Phase 切换是否伴随场地变化(变形/破坏/扩展)?
- [ ] 是否有时间维度门控(Active Gauge / 倒计时 / 节奏锚点)?
- [ ] hazard 触发条件 + 伤害值是否明确?
- [ ] 玩家可利用场地的方式(钩拽/坠落/召唤/破坏)?
