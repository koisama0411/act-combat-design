    📋 文档协作使用说明

    本文档可在浏览器内直接编辑 / 划除 / 备注 / 新增。
    ⚠️ 修改只临时缓存在当前标签页:F5 刷新会保留，但关闭标签页 / 浏览器即丢失 — 必须点「📤 导出修改版」才能永久保存。

    1. 改完点左侧 📤 导出修改版 → 下载一份带你全部修改的 HTML 文件（文件名带日期 + 你的名字）
    2. 把这份文件发回给设计师 / 团队
    3. 收件人打开后，文档自动加载你的修改作为新基线
    资源清单的两个协作列（都支持文本/链接/截图/视频，反复粘贴可在一格放多个）：
    · 📚 参考 — 策划填：给美术/特效的参考材料（原作截图、参考视频、风格示例）
    · 📎 产出/协作 — 美术 / 特效 / 音效 / UI填：你的产出链接 / 状态 / 截图 / 视频
    两列都点进去直接填，自动缓存，无需点确认按钮。⚠️ 不要碰其他列（动作名 / 用途 等是设计师字段）。整体意见写每章末右侧 📝 评审备注 区。
    💡 协作列 / 评审备注支持的内容:
    • 纯文本 / 链接 — 直接打字或粘贴 URL
    • 截图 / 动图 GIF — Ctrl+V 粘贴 / 拖拽进来，浏览器自动 base64 内嵌
    • 本地视频 (mp4/webm/mov) — 粘贴或拖拽 ≤ 200 MB 的视频，自动内嵌播放器（>50MB 会二次确认；超 5MB 自动走 IndexedDB 大文件存储）
    • 一格可贴多个 — 同一格反复粘贴 / 拖入，媒体会依次排列；每个媒体 hover 有 ⛶ 放大 + ✕ 删除
    • 视频直链 — 粘贴 .mp4 / .webm URL 自动转 <video> 渲染
    • B站 / YouTube 链接 — 粘贴自动嵌入播放器 iframe
    ⚠️ 大文件优先贴链接（网盘 / B站 / 内部图床），浏览器存储约 5 MB 上限，大量 base64 易爆配额导致保存失败。
